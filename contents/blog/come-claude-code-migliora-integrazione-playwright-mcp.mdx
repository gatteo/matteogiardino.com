[L'automazione del browser](https://matteogiardino.com/blog/automazione-browser-openclaw) di solito richiede script rigidi e lineari. Ma integrando Claude Code con Playwright attraverso il Model Context Protocol (MCP), puoi istruire un agente AI a navigare e testare le app web localmente, direttamente dal tuo terminale.

Se stai costruendo agenti AI o gestendo test end-to-end, collegare un LLM direttamente a un'istanza reale del browser è incredibilmente potente.

## Cos'è il Model Context Protocol (MCP)?

Il Model Context Protocol (MCP) è un modo standardizzato per i modelli AI di accedere a strumenti e ambienti locali. Pensalo come un ponte tra un LLM e il tuo sistema. Invece di avere un'AI isolata, un server MCP espone delle capacità - come leggere file o controllare un browser - che il modello può usare in modo sicuro.

## Perché Collegare Claude Code con Playwright?

Claude Code è l'agente CLI di Anthropic. Di base, può leggere il tuo codice e scrivere file. Ma aggiungendo un server MCP per Playwright, dai a Claude Code la capacità di:
- Aprire le web app locali durante lo sviluppo.
- Cliccare pulsanti, compilare form e fare screenshot.
- Verificare visivamente se una modifica alla UI ha rotto il layout.

Passi dallo scrivere rigidi script `page.click()` a dare comandi di alto livello come: "Apri localhost:3000 e controlla se il form di login funziona."

## Guida Pratica al Setup

Far comunicare Claude Code con Playwright richiede il setup di un server MCP che esponga le API Node di Playwright.

Prima di tutto, assicurati di avere Claude Code installato. Poi, inizializza un server MCP base per Playwright:

```bash
npx create-mcp-server playwright-mcp
cd playwright-mcp
npm install playwright
```

Dovrai esporre le azioni standard di Playwright (naviga, clicca, scrivi, screenshot) come strumenti MCP. Una volta avviato il server, configura Claude Code per connettersi ad esso. Modifica la configurazione di Claude Code per includere il nuovo endpoint:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "node",
      "args": ["/path/to/playwright-mcp/build/index.js"]
    }
  }
}
```

Riavvia Claude Code e ora avrà accesso agli strumenti di automazione del browser.

## Esempio Reale: Test E2E Autonomi

Con l'integrazione completata, puoi avviare una sessione di Claude Code nel terminale e chiedere direttamente:

> "Naviga su http://localhost:8080. Clicca il pulsante 'Inizia', compila il campo email con 'test@example.com' e fai uno screenshot del risultato."

Claude Code usa il ponte MCP per eseguire questi comandi Playwright in sequenza. Se un selettore fallisce, il modello riceve il contesto dell'errore e può riprovare con un selettore diverso - un livello di resilienza che gli script tradizionali non hanno.

## Limiti e Considerazioni sulla Sicurezza

Far girare agenti web autonomi in locale comporta dei rischi. L'agente ha pieno controllo sulla sessione del browser. Se gli ordini di eseguire azioni su siti live e autenticati, potrebbe attivare rate limit o compiere azioni distruttive.

Usa sempre ambienti di test dedicati o contesti isolati. Inoltre, la velocità di elaborazione visiva di Claude Code dipende dalle tue quote API, quindi test visivi pesanti potrebbero esaurire rapidamente i limiti del tuo piano Anthropic.
