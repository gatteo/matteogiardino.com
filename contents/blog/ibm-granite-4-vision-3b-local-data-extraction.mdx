I recently needed to extract structured data from hundreds of corporate PDFs but couldn't send them to OpenAI or Anthropic due to strict privacy constraints. **IBM Granite 4.0 Vision 3B** is an ultra-compact, open-source vision-language model (VLM) designed specifically to convert scanned documents, tables, and charts into JSON and CSV locally. After stress-testing it on my office servers, here is how to install and use it.

## What is IBM Granite 4.0 Vision 3B

IBM Granite 4.0 3B Vision is a hybrid model that combines a base LLM (the 3.5B Granite 4.0 Micro) with a 0.5B parameter visual LoRA adapter. This architecture provides an incredibly lightweight model (under 4 billion parameters total) that understands complex document layouts with the accuracy of much larger models.

It was heavily trained on ChartNet, a massive dataset containing over a million charts and tables. Consequently, it excels where smaller models usually fail: the spatial understanding of receipts, invoices, and financial reports.

## Data Extraction: Tables, Charts, and KVPs

The true power of Granite Vision 3B lies in its output formatting. It can be instructed to return data in the following formats:

1. **Chart2CSV and Chart2Summary**: Extracts numerical values directly from the bars and lines of a scanned chart image.
2. **Table Extraction**: Transforms complex, nested tables into HTML, OTSL, or structured JSON.
3. **KVP (Key-Value Pair)**: Extracts semantic values based on field names, regardless of where they are positioned in the document layout.

This makes it the perfect building block for automated pipelines with AI agents.

## How to Install Granite 4.0 Vision Locally

There are several ways to spin up the model. The most performant method for production inference is using **sglang** via Docker.

### Starting with Docker

If you have a GPU and NVIDIA drivers installed, you can launch an OpenAI-compatible API server in seconds:

```bash
docker run --gpus all \
    --shm-size 32g \
    -p 30000:30000 \
    -v ~/.cache/huggingface:/root/.cache/huggingface \
    --ipc=host \
    lmsysorg/sglang:latest \
    python3 -m sglang.launch_server \
        --model-path "ibm-granite/granite-4.0-3b-vision" \
        --host 0.0.0.0 \
        --port 30000
```

### Making an API Call

Once the server is running, you can send images and extract text just like you would with GPT-4 Vision:

```bash
curl -X POST "http://localhost:30000/v1/completions" \
  -H "Content-Type: application/json" \
  --data '{
    "model": "ibm-granite/granite-4.0-3b-vision",
    "prompt": "Extract the table in this image to JSON.",
    "max_tokens": 1024,
    "temperature": 0.1
  }'
```

*(Note: for full multimodal calls with sglang you will need to pass the base64 image in the compatible messages format; this is just a quick ping example).*

## Frequently Asked Questions

### How does IBM Granite 4.0 3B Vision work?
It is a VLM that uses a 3.5B base LLM and a 0.5B LoRA vision adapter. It processes document images and extracts structured data like JSON or CSV.

### Can I run Granite Vision 4.0 on CPU?
Yes, you can use frameworks like llama.cpp to run quantized versions on a CPU, though optimal performance is achieved with a GPU or accelerators like Apple Silicon.

### Is it suitable for non-English languages?
Yes. Granite is trained to be multilingual and can reliably extract concepts and fields from Italian, French, or Spanish documents.
