## Requirements: Ollama and compatible models

To trick the tool into thinking it is talking to the cloud, you need a local model with excellent *tool-use* and coding capabilities. Not all models perform well when handling the complex system prompts this agent relies on.

The two best current models for this purpose are:
- `qwen2.5-coder:32b` (Excellent tool management and context retention).
- `deepseek-coder-v2` (Extremely fast for standard tasks).

Make sure you have Ollama installed, and pull your chosen model:

```bash
ollama pull qwen2.5-coder:32b
```

## The trick: Redirecting the API

The CLI natively looks for the cloud address. There is no direct setting to tell it "use local server instead." The elegant workaround is to redirect its network calls. 

You have two options:
1. Use **LiteLLM** as a local proxy, which translates the format into the format Ollama expects on the fly.
2. Use a dedicated fork or bridge script that simulates the endpoint.

In this guide, we will use LiteLLM because it is the industry standard for model routing and is incredibly stable.

## Practical setup step-by-step

### 1. Start LiteLLM with the Ollama model

First, install LiteLLM via Python. As discussed in [How MetaClaw and Ollama Make OpenClaw Smarter](/blog/metaclaw-ollama-make-openclaw-smarter), using local tools effectively requires a solid proxy. Start it in proxy mode, pointing to the Ollama model you just pulled:

```bash
pip install litellm
litellm --model ollama/qwen2.5-coder:32b --port 4000
```
The proxy is now listening on port 4000 and is ready to receive and translate calls intended for the cloud.

### 2. Configure variables for the system

Now you need to tell the tool to target your local server instead of the cloud. Open a new terminal and set the base API URL:

```bash
export ANTHROPIC_BASE_URL="http://localhost:4000"
export ANTHROPIC_API_KEY="sk-fake-key"
```

### 3. Launch the agent

Finally, run the tool. (LiteLLM will map the request to your local model automatically):

```bash
claude
```
Now all your terminal requests will be processed locally by your Mac or PC!

## FAQ

### Is it legal to use Claude Code without an API Key?
Yes. You are simply using the open-source (or locally downloaded) CLI interface and redirecting traffic to your private server. You are not hacking their systems; you are just not calling them.

### What is the best Ollama model for coding?
Currently, `qwen2.5-coder:32b` and its variations offer the best balance between size, logical capabilities, and the precision required for the tool-use patterns of autonomous agents.

### Can I use Claude Code completely offline?
Absolutely. Once the model is downloaded to Ollama and LiteLLM is running locally, the entire workflow happens on your machine (localhost). You can literally turn off your Wi-Fi and keep coding with AI assistance.
