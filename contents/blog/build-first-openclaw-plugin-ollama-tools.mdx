## Development Prerequisites for Local Plugins

To follow this tutorial, you need a specific local setup. Ensure you have the following installed and running on your machine:

1. Node.js (version 20 or later).
2. A local Ollama instance running locally. For this guide, I use `qwen2.5-coder:32b`.
3. The latest `@openclaw/core` package installed via npm.
4. An IDE like VS Code or Cursor.

If you have never configured a local AI agent before, you should first read my guide on [how to install OpenClaw with Ollama](/blog/how-to-install-openclaw-ollama-2026-guide) to get the basic environment running properly. This will save you a lot of debugging time.

## Plugin Architecture Explained

An OpenClaw plugin simply registers a toolkit by exposing a standard interface for agents. The plugin acts as a bridge. It allows the agent to dynamically discover which functions it can invoke. For example, reading files, searching the web, or querying an internal database.

When you are building your first openclaw plugin with ollama tools, you must define the schema for each tool. Ollama now supports native tool calling (function calling). This means the local LLM understands JSON schemas and can output structured JSON to trigger your plugin's code. This is a massive shift for local AI capabilities. 

## Writing the Plugin Code

Here is a comprehensive example. We will define a custom plugin that provides a calculator tool to the agent. This code defines the plugin class and registers the schema for Ollama to parse.

```typescript
import { OpenClawPlugin, ToolContext } from '@openclaw/core';

export class OllamaToolsPlugin implements OpenClawPlugin {
  name = 'ollama-calculator-tools';
  
  getTools() {
    return [
      {
        name: 'calculate_sum',
        description: 'Calculates the sum of two numbers',
        parameters: {
          type: 'object',
          properties: {
            a: { type: 'number', description: 'First number' },
            b: { type: 'number', description: 'Second number' }
          },
          required: ['a', 'b']
        }
      }
    ];
  }

  async execute(toolName: string, args: any, context: ToolContext) {
    if (toolName === 'calculate_sum') {
      const result = args.a + args.b;
      return { success: true, value: result };
    }
    throw new Error(`Tool ${toolName} not found.`);
  }
}
```

This simple plugin architecture can scale. You can add more tools to the `getTools` array. Just make sure the descriptions are very clear, as Ollama relies on these descriptions to decide when to call the tool.

## Connecting Ollama and Testing the Tool Call

After writing the plugin, you must connect it to the OpenClaw agent instance. You configure the agent to use your local Ollama server endpoint.

```typescript
import { Agent } from '@openclaw/core';
import { OllamaToolsPlugin } from './OllamaToolsPlugin';

async function runAgent() {
  const agent = new Agent({
    model: 'qwen2.5-coder:32b',
    provider: 'ollama',
    baseUrl: 'http://localhost:11434/api',
    plugins: [new OllamaToolsPlugin()]
  });

  const response = await agent.chat('What is 452 plus 789?');
  console.log(response);
}

runAgent();
```

When you execute this code, the agent sends the prompt to Ollama. Ollama sees the `calculate_sum` tool in the system prompt. It outputs a JSON tool call. OpenClaw intercepts this call, runs your `execute` function, and returns the result to Ollama. Finally, Ollama formulates the natural language answer. This cycle is fast, private, and fully local.

## FAQ

### What models support tool calling in Ollama?
As of late 2025 and 2026, models like Qwen 2.5, Llama 3.3, and Mistral all feature robust, native tool calling support directly inside Ollama.

### Is OpenClaw free to use?
Yes. OpenClaw is an open-source framework designed for building autonomous agents without vendor lock-in.

### Can I connect databases using plugins?
Absolutely. You can build a plugin that holds a PostgreSQL connection pool and exposes a `query_db` tool to the agent. Just ensure you implement strict schema validation and read-only credentials for safety.
