## Il Problema della Privacy e la Ricerca Documenti Offline

Molti tutorial danno per scontato che tu possa caricare PDF, wiki aziendali e bilanci su API cloud. Nella mia esperienza, questo è inaccettabile per molti CTO. Secondo un sondaggio del 2026, oltre il 65% delle aziende vieta l'upload di codice proprietario su modelli pubblici.

Spostando l'intera pipeline RAG in locale, ottieni tre vantaggi:
- **Zero data leakage**: I tuoi documenti non lasciano mai la tua macchina. Questo è vitale per la compliance.
- **Nessun costo ricorrente**: Una volta configurato l'hardware, l'inferenza è gratis.
- **Funzionamento air-gapped**: Funziona perfettamente anche senza connessione internet.

Ollama rende semplicissimo eseguire i modelli. OpenClaw fornisce il framework perfetto per orchestrare la ricerca. Se sei nuovo a questo mondo, ti consiglio di leggere la mia guida sull'[architettura e i cicli degli agenti OpenClaw](/blog/flussi-lavoro-agenti-ollama-openclaw).

## Lo Stack RAG Locale per il 2026

La nostra architettura offline si basa su tre componenti:
1. **Ollama**: Funge da provider LLM locale. Usiamo un modello veloce come `llama3` per generare testo e `nomic-embed-text` per creare i vettori.
2. **Local Vector Store**: Usiamo ChromaDB. È semplice, funziona in Python e salva i dati direttamente su disco senza configurazioni complesse.
3. **OpenClaw**: L'orchestratore. Prende le domande dell'utente, cerca nel database e genera la risposta.

## Step 2: Configurare il Database Vettoriale

Ora serve uno script per ingerire i documenti, convertirli in vettori e salvarli in locale.

```python
import chromadb
import requests
import json

client = chromadb.PersistentClient(path="./local_rag_db")
collection = client.get_or_create_collection(name="internal_docs")

def get_embedding(text):
    response = requests.post('http://localhost:11434/api/embeddings',
                             json={'model': 'nomic-embed-text', 'prompt': text})
    return response.json()['embedding']

documents = [
    "OpenClaw è un framework modulare per agenti AI.",
    "Ollama ti permette di eseguire LLM in locale facilmente."
]

for i, doc in enumerate(documents):
    embedding = get_embedding(doc)
    collection.add(
        embeddings=[embedding],
        documents=[doc],
        ids=[f"doc_{i}"]
    )
print("Ingestione completata!")
```

Questo codice è semplice ed efficace. Puoi eseguirlo offline senza problemi. Consiglio di usarlo a piccoli blocchi se hai migliaia di PDF.

## Step 3: Configurare l'Agente OpenClaw

Ora creiamo un agente OpenClaw e gli diamo uno strumento per cercare nel nostro ChromaDB locale. Per approfondire gli strumenti, puoi leggere la mia guida su come [creare un server MCP personalizzato per OpenClaw](/blog/8-migliori-plugin-openclaw-2026).

```typescript
import { Agent, Tool } from 'openclaw';
import { ChromaClient } from 'chromadb';

const db = new ChromaClient({ path: "./local_rag_db" });

const searchDocsTool = new Tool({
  name: "search_internal_docs",
  description: "Cerca nel database locale i documenti interni.",
  execute: async (query) => {
    // Converte la query in embedding
    const collection = await db.getCollection("internal_docs");
    const results = await collection.query({
        queryTexts: [query],
        nResults: 2
    });
    return results.documents[0].join("\n");
  }
});

const ragAgent = new Agent({
  model: 'ollama/llama3',
  tools: [searchDocsTool],
  systemPrompt: "Sei un assistente utile. Usa sempre il tool search_internal_docs per rispondere alle domande basandoti sui documenti locali."
});
```

L'agente userà questo strumento per trovare contesto prima di generare una risposta. Questa è l'essenza di un sistema RAG locale: veloce e privato.

## FAQ

### Posso eseguirlo su un normale laptop?

Sì. Puoi far girare `llama3` e il modello vettoriale su qualsiasi laptop recente con almeno 16GB di RAM. I Mac con Apple Silicon sono eccezionali per questo.

### ChromaDB è l'unica opzione?

No. Puoi usare Qdrant, Milvus o pgvector. Ho scelto ChromaDB perché non richiede setup complicati e gira nativamente in Python.

### Come aggiorno i documenti?

Puoi scrivere un piccolo script che controlla una cartella e aggiorna il database quando un file cambia, calcolando il nuovo vettore.
