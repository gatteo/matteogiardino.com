Passavo metà del mio tempo a fare copia-incolla di errori del terminale o snippet di VS Code per dare contesto agli agenti AI. Screenpipe cambia tutto registrando schermo e microfono 24/7 in un database locale. Dopo averlo collegato a OpenClaw e Ollama, i miei agenti possono "vedere" istantaneamente cosa stavo guardando 10 minuti fa, completamente offline.

## Il Problema del Contesto

Gli agenti AI falliscono quando non hanno contesto. Se chiedi a un agente di "scrivere un'email basata sulla riunione Zoom appena terminata", un setup standard si blocca. Non ha idea di cosa sia successo sul tuo schermo o di cosa sia stato detto.

O scrivi un prompt enorme per spiegare tutto, o usi un tool cloud proprietario che registra tutto ciò che fai e lo invia a server che non controlli. Nessuna delle due opzioni è l'ideale per la privacy o la velocità operativa.

## Cos'è Screenpipe?

Screenpipe è uno strumento open-source che acquisisce continuamente il tuo schermo e il tuo microfono. A differenza delle alternative commerciali, archivia tutti questi dati localmente in un database SQLite e li espone tramite un'API locale. Elabora l'OCR e lo speech-to-text direttamente sulla tua macchina.

Collegando questa API locale a OpenClaw tramite MCP (Model Context Protocol), dai al tuo agente AI una memoria perfetta della tua giornata digitale.

## L'Architettura

Quando OpenClaw esegue un modello locale (come Llama 3 via Ollama), ha bisogno di un modo per interrogare Screenpipe.

1. **Screenpipe** registra schermo (OCR) e microfono (STT).
2. **Server MCP di OpenClaw** espone un'API per cercare le ultime ore di attività.
3. **Agente OpenClaw** interroga questo strumento ogni volta che ha bisogno di contesto.

## Configurare l'Integrazione

### 1. Installare Screenpipe

Per prima cosa, avvia Screenpipe sulla tua macchina. Assicurati che abbia i permessi corretti (Registrazione Schermo e Microfono).

```bash
curl -sL https://screenpi.pe/install.sh | bash
screenpipe start
```

### 2. Configurare la Skill MCP di OpenClaw

Crea una nuova skill in OpenClaw che interroga l'API locale di Screenpipe (porta predefinita `3030`).

```json
{
  "name": "screenpipe_context",
  "description": "Cerca nella cronologia dello schermo l'attività recente.",
  "command": "curl -s 'http://localhost:3030/search?q={query}&limit=5'"
}
```

Ora, quando chiedi al tuo agente "Qual era l'errore in VS Code 5 minuti fa?", lui cerca automaticamente su Screenpipe, legge il testo OCR e ti dà la soluzione.

## Privacy vs Utilità

L'unico modo sicuro per avere una memoria AI "sempre attiva" è eseguirla localmente. Combinando Ollama con Screenpipe, i dati del tuo schermo non lasciano mai il tuo MacBook. Ottieni l'utilità di un secondo cervello senza caricare le tue chiavi private o gli appunti delle riunioni sul cloud.

## Domande Frequenti

### Screenpipe rallenta il mio Mac?
Screenpipe è ottimizzato per Apple Silicon e funziona in background consumando in genere meno del 10% della CPU.

### Posso filtrare cosa viene registrato?
Sì, puoi configurare Screenpipe per ignorare app specifiche come password manager o browser in incognito.

### Funziona su Windows?
Sì, Screenpipe supporta Windows, macOS e Linux, rendendolo un'ottima soluzione multipiattaforma per il contesto locale.

## Conclusione

Integrare Screenpipe con OpenClaw rimuove l'attrito dei prompt manuali. I tuoi agenti diventano partner proattivi piuttosto che chatbot reattivi. Se vuoi approfondire i setup locali, guarda la mia guida su come eseguire l'AI in locale.
