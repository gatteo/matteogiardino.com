Lookspan è la piattaforma open source rilasciata nel 2026 che ti permette di monitorare e tracciare le chiamate LLM dei tuoi agenti AI interamente in locale. Gestire agenti AI in locale è ottimo per la privacy, ma tracciare i log e l'uso dei token diventa un incubo senza gli strumenti giusti. Lookspan risolve esattamente questo problema: elaborando oltre 1.000 span al secondo con un impatto minimo sulle performance, è una piattaforma di osservabilità pensata per i workload degli LLM che puoi far girare interamente sulla tua macchina. Dopo aver provato vari sistemi cloud costosi come LangSmith (che spesso superano i 200$ al mese per carichi di media entità), ho deciso di integrare Lookspan nel mio stack locale. In questo articolo ti mostro come configurarlo insieme a OpenClaw e Ollama (se non sai come fare, dai un'occhiata alla mia [guida su come usare Ollama con OpenClaw](/blog/usare-ollama-con-openclaw-it)).

## Cos'è Lookspan e perché serve agli agenti AI

Lookspan è uno strumento di osservabilità per agenti AI che raccoglie, visualizza e analizza i trace delle chiamate ai modelli linguistici. A differenza delle soluzioni SaaS tradizionali che richiedono di inviare i dati dei tuoi prompt su server esterni, Lookspan è progettato per essere *local-first*.

Quando si costruiscono agenti autonomi, si eseguono decine di chiamate LLM in background: generazione, validazione, parsing JSON e retry. Se qualcosa si rompe o i token consumati esplodono, debuggare tramite console log è praticamente impossibile. Lookspan intercetta questi flussi fornendo una dashboard chiara con la durata di ogni span, l'input esatto, l'output generato e il conteggio dei token.

## Come configurare Lookspan in locale

Installare Lookspan è rapido se utilizzi Docker. La piattaforma è composta da un server per la raccolta dei trace (tipicamente compatibile con OpenTelemetry) e una web UI.

### Step 1: Avviare Lookspan con Docker

Il metodo più pulito per eseguire Lookspan è tramite Docker Compose. Crea un file `docker-compose.yml` con la configurazione ufficiale:

```yaml
version: '3'
services:
  lookspan:
    image: lookspan/lookspan:latest
    ports:
      - "8080:8080" # Web UI
      - "4317:4317" # OTLP gRPC
      - "4318:4318" # OTLP HTTP
```

Quindi lancia il container:

```bash
docker-compose up -d
```

Una volta avviato, la dashboard sarà disponibile su `http://localhost:8080`.

### Step 2: Integrazione con OpenClaw

Se utilizzi OpenClaw per gestire i tuoi agenti, puoi configurare facilmente l'invio dei dati verso il nodo Lookspan appena creato. Essendo compatibile con lo standard OpenTelemetry, basta impostare le variabili d'ambiente corrette prima di lanciare il tuo agente:

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4318"
export OTEL_TRACES_EXPORTER="otlp"
export OTEL_METRICS_EXPORTER="none"
```

A questo punto, ogni volta che il tuo agente OpenClaw effettuerà una chiamata a Ollama o ad altri provider locali, la traccia completa sarà visibile nella dashboard di Lookspan.

## Cosa è andato storto le prime volte

Uno dei problemi principali che ho incontrato riguardava i conflitti di porta con altri stack locali. Inizialmente avevo un'istanza di Jaeger attiva sulla porta 4317. Quando provavo a lanciare Lookspan, i trace venivano droppati. La soluzione è stata banale: fermare i vecchi container di tracing e mappare Lookspan in modo esclusivo su OTLP, oppure cambiare i binding delle porte nel `docker-compose.yml` e aggiornare l'endpoint `OTEL_EXPORTER_OTLP_ENDPOINT`.

Inoltre, ricorda che i log degli agenti possono crescere rapidamente. Se usi Lookspan in un ambiente di testing continuo, imposta una policy di data retention per evitare di saturare il disco del tuo Mac Mini o server.

## Risultato: Osservabilità totale senza cloud

Da quando ho inserito Lookspan nel mio flusso di lavoro, il debugging delle pipeline AI locali è diventato istantaneo. Non devo più scrollare infinite righe di terminale per capire quale prompt ha fatto deragliare un agente. Questa soluzione chiude perfettamente il cerchio: modelli eseguiti in locale con Ollama, orchestrazione gestita con OpenClaw e telemetria sicura con Lookspan. Nessun dato lascia mai la macchina.

## Domande frequenti

### Cos'è Lookspan?
Lookspan è una piattaforma open source per l'osservabilità degli agenti AI, specializzata nel tracciamento (tracing), logging e monitoraggio delle chiamate ai modelli LLM in ambienti locali.

### Quali sono le alternative a Lookspan?
Le alternative cloud più note sono LangSmith e Phoenix, mentre per il mondo open source si usano spesso combinazioni di OpenTelemetry standard con Jaeger o SigNoz, anche se meno specifiche per i prompt degli LLM.

### Posso usare Lookspan con Ollama?
Sì. Lookspan agisce a livello di orchestratore (es. OpenClaw o LangChain) intercettando le chiamate API dirette a Ollama, permettendoti di monitorare l'input, l'output e la latenza dei tuoi modelli eseguiti in locale.

## Conclusione

L'osservabilità è l'anello mancante per passare da semplici script AI a veri e propri agenti autonomi affidabili. Lookspan dimostra che non è necessario dipendere da costose soluzioni SaaS o sacrificare la privacy dei propri dati per avere dashboard professionali e tracing avanzato.
