## What is DramaBox 3.3B Local TTS?

DramaBox 3.3B local TTS is an audio-diffusion transformer built on top of LyraX LTX2. It uses flow matching conditioned on Gemma 3 (12B) text embeddings. 

This architecture combines a diffusion transformer backbone with an audio variational autoencoder. This setup translates the hidden space representations into audible sound, while a built-in vocoder manages the timing and pauses. The reliance on Gemma 3 is what allows DramaBox 3.3B local TTS to deeply understand complex context and emotional cues.

If you have used tools like Fish Audio before (read more about [Fish Audio local setup](/blog/fishaudio-s2-pro-local-voice-cloning)), you know that local AI voices are getting incredibly realistic in 2026. But DramaBox takes this to another level by introducing theatrical control, letting you manage the exact way your AI characters speak. 

## Local Setup and VRAM Requirements

Running DramaBox 3.3B local TTS locally requires a solid GPU. In my tests, VRAM consumption sat just over 16 GB during generation and playback, meaning an RTX 4090, RTX 6000, or a Mac with 32GB+ Unified Memory is ideal. 

You can set it up via a standard Python virtual environment:

```bash
git clone REPO_URL dramabox
cd dramabox
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
python demo.py
```

This launches a Gradio interface on `localhost:7860`. If you are already running local models with Ollama, this setup will feel very familiar. ## Prompt as a Performance Script

The most innovative feature of DramaBox 3.3B local TTS is how it handles text inputs. It treats the prompt as a full performance script. 

Dialogue placed inside quotes is spoken aloud, while everything outside the quotes acts as a stage direction. This allows you to shift from a whisper to a shout mid-sentence:

```text
<voice softens, short pause>
"I can't believe it."
<voice rises with anger>
"Why didn't you tell me?!"
<sigh>
```

You are not just converting text to speech; you are directing a voice actor. Keep your stage directions concise and only describe things that affect delivery. The more specific you are with your stage directions, the better the model performs. You can layer multiple emotions, create suspenseful pauses, or add realistic artifacts like sighs and short laughs to humanize the delivery.

If you are a developer looking to integrate this, think of the stage directions as parameters you can inject dynamically. For example, if you build an AI agent that monitors system logs, you can program it to output a calm voice for routine updates and an urgent voice for critical errors. 

## Voice Cloning Limitations and Quality

DramaBox 3.3B local TTS allows you to pass a 10-second reference audio file to clone a specific voice. However, voice cloning is currently not its strongest suit. 

In my tests, the model successfully captured the *expression* and intent of the reference, but the actual timbre (the unique sound of the voice) tended to drift. Additionally, on longer scripts with complex emotional transitions, the model sometimes hallucinates—skipping words or rushing through lines.

## FAQ

**How much VRAM do I need for DramaBox 3.3B local TTS?**
You need around 16 GB of VRAM to generate audio smoothly, making an RTX 4090, RTX 6000, or similar high-end GPU highly recommended for local inference.

**Can DramaBox clone my voice?**
Yes, it supports 10-second reference clips for voice cloning, though the timbre matching is currently less stable than its emotional expression capabilities. Future fine-tunes will likely improve this.

**How do I add pauses in DramaBox?**
You can control pacing and add pauses by writing stage directions outside of quotes, such as `<short pause>` or `<deep breath>`. The model interprets these as literal performance cues.
