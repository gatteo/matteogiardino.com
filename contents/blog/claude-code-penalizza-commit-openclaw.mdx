All'inizio del 2026, ho notato un picco anomalo nei costi dei token e strani rifiuti durante l'utilizzo di Claude Code. Dopo aver analizzato i log e letto un thread recente su Hacker News, la realtà è emersa chiaramente: è emerso che Claude Code penalizza i commit che menzionano OpenClaw nei repository che menzionano "OpenClaw" nei loro commit git. Se la CLI rileva il framework concorrente nella sua fase di raccolta del contesto, si rifiuta di eseguire determinati loop o gonfia inutilmente il costo del contesto.

Si tratta di una rivelazione enorme per gli sviluppatori che costruiscono su framework AI locali e sottolinea il crescente attrito tra strumenti proprietari e l'ecosistema open-source.

## La Scoperta su Hacker News

Il problema è stato segnalato per la prima volta su Hacker News (una fonte autorevole per l'ecosistema tech), quando alcuni sviluppatori hanno notato che Claude Code falliva su compiti banali. Isolando le variabili, si è scoperto che bastava aggiungere la stringa "OpenClaw" a un messaggio di commit recente o avere una directory chiamata `openclaw/` nel workspace per innescare un comportamento da shadow-ban.

Claude Code legge la cronologia git recente e la struttura dei file come contesto prima di eseguire il suo ciclo di pianificazione. Sembra che Anthropic abbia istituito un filtro o una direttiva nascosta nel prompt di sistema che considera i riferimenti a OpenClaw come una violazione.

## Come Funziona la Penalizzazione

Quando esegui un comando tramite la CLI di Claude Code, lo strumento non si limita a leggere il file corrente. Scansiona i tuoi `.git/logs`, i messaggi di commit recenti e il layout del workspace per costruire una consapevolezza situazionale.

Se incontra la parola chiave "OpenClaw", in genere accadono due cose:
1. **Rifiuti:** Il modello emette una risposta preimpostata affermando di non poter assistere con "framework agentici non supportati o esterni", abbandonando completamente il compito.
2. **Picchi di Contesto:** Nei casi in cui non si rifiuta, gli utenti segnalano che la CLI entra in un loop confusionario, estraendo ripetutamente gli stessi log git e aumentando i costi dei token per la sessione prima di fallire.

## L'Impatto Economico sugli Sviluppatori

Il "picco di contesto" è particolarmente insidioso perché ti addebita costi extra senza fornire valore. Claude Code è fatturato a token. Quando l'agente si blocca in un loop cercando di conciliare un prompt di sistema che vieta OpenClaw con un prompt dell'utente che gli chiede di fare debug, brucia migliaia di token in pochi secondi.

Per i team che eseguono flussi di lavoro automatizzati o usano Claude Code per lo sviluppo quotidiano, questa penalità nascosta può portare a bollette sorprendenti a fine mese.

## Perché Anthropic Potrebbe Farlo

Anche se Anthropic non ha commentato ufficialmente, il consenso della community è chiaro: OpenClaw rappresenta una minaccia significativa per i flussi di lavoro proprietari e vincolati. [OpenClaw](/blog/architettura-openclaw-loop-agentici) consente agli sviluppatori di indirizzare i compiti agentici attraverso modelli locali come Ollama, bypassando completamente le API di Anthropic per i compiti più semplici.

Rendendo Claude Code ostile ai workspace OpenClaw, Anthropic potrebbe tentare di forzare gli sviluppatori a scegliere tra la loro CLI e gli agenti locali open-source. Tuttavia, nello spazio degli strumenti per sviluppatori, l'attrito artificiale di solito si ritorce contro.

## Soluzioni per Aggirare il Filtro

Non devi abbandonare Claude Code se sei un utente OpenClaw. Devi solo nascondere le parole chiave dalla fase di raccolta del contesto di Claude.

### 1. Usa .claudesignore

La soluzione più semplice è impedire a Claude Code di vedere le directory o i log in cui viene menzionato OpenClaw. Aggiungi le tue cartelle specifiche per OpenClaw a un file `.claudesignore` nella root del tuo progetto:

```text
# .claudesignore
openclaw/
*.openclaw
```

### 2. Pulisci i tuoi Log Git

Poiché Claude Code legge i commit recenti, evita di usare l'esatta stringa "OpenClaw" nei messaggi. Usa un'abbreviazione come `oc-framework` o `agente-locale`.

Se hai già commit problematici nella tua cronologia recente, puoi unirli (squash) o limitare quanto indietro guarda Claude Code restringendo i permessi di accesso ai file.

## L'Effetto Dissuasivo sugli Agenti Open-Source

Questo comportamento stabilisce un precedente pericoloso. Se i fornitori di modelli iniziano a usare le loro CLI come armi per soffocare le alternative open-source, gli sviluppatori perderanno l'interoperabilità che rende l'ecosistema AI così potente. Costruire sistemi resilienti significa affidarsi a standard aperti, non a strumenti black-box che ti penalizzano silenziosamente.

## Domande frequenti

### Claude Code cancella i file OpenClaw?

No. Claude Code non elimina o modifica in modo dannoso i tuoi file OpenClaw. Si rifiuta semplicemente di operare correttamente o spreca token quando rileva la parola chiave nella sua finestra di contesto.

### Questo influisce sull'interfaccia web standard di Claude?

Attualmente, questo comportamento sembra isolato alla CLI di Claude Code, che estrae attivamente il contesto di sistema locale come i commit git. L'interfaccia web opera in modo diverso e non scansiona l'ambiente locale.

### Posso ancora usare OpenClaw insieme a Claude Code?

Sì, a patto di utilizzare soluzioni alternative come `.claudesignore` per nascondere alla CLI la presenza di directory OpenClaw ed evitare di usare la parola chiave nella cronologia dei commit.

## Conclusione

La penalizzazione silenziosa sui commit OpenClaw è un frustrante promemoria dei rischi del vendor lock-in. Mantieni i tuoi flussi di lavoro modulari, usa agressivamente `.claudesignore` e mantieni sempre la capacità di sostituire i tuoi strumenti AI sottostanti.
