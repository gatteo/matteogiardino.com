Questo approccio è fondamentale per chi lavora su progetti aziendali sensibili o chi semplicemente vuole fare prototipazione intensiva senza tenere d'occhio il portafoglio.

## Requisiti: Ollama e modelli compatibili

Per far credere al tool di parlare con il cloud, specialmente dopo aver imparato [come configurare OpenClaw con Ollama: guida pratica per agenti AI locali](/blog/configurare-openclaw-ollama-agenti-ai-locali), hai bisogno di un modello locale con eccellenti capacità di *tool-use* e programmazione. Non tutti i modelli funzionano bene con i prompt complessi dell'agente.

I due migliori modelli attuali per questo scopo sono:
- `qwen2.5-coder:32b` (Eccellente gestione dei tool e contesto).
- `deepseek-coder-v2` (Molto veloce per task standard).

Assicurati di aver installato Ollama e di scaricare il modello:

```bash
ollama pull qwen2.5-coder:32b
```

## Il trucco: Redirigere le API

Il tool cerca nativamente un indirizzo cloud. Non esiste un'impostazione diretta per dirgli "usa il server locale". La soluzione elegante consiste nel redirigere le sue chiamate. 

Hai due opzioni:
1. Usare **LiteLLM** come proxy locale, che traduce "al volo" il formato nel formato che Ollama si aspetta.
2. Sfruttare un fork o script bridge che simula l'endpoint.

In questa guida useremo LiteLLM perché è lo standard industriale per il routing dei modelli ed è estremamente stabile.

## Configurazione pratica passo-passo

### 1. Avvia LiteLLM con il modello Ollama

Per prima cosa, installa LiteLLM via Python e fallo partire in modalità proxy, puntando al modello Ollama che hai scaricato:

```bash
pip install litellm
litellm --model ollama/qwen2.5-coder:32b --port 4000
```
Il proxy ora ascolta sulla porta 4000 ed è pronto a ricevere e tradurre le chiamate destinate al cloud.

### 2. Configura le variabili per il sistema

Ora devi dire al programma di puntare al tuo server locale anziché al cloud. Apri un nuovo terminale e imposta l'URL base:

```bash
export ANTHROPIC_BASE_URL="http://localhost:4000"
export ANTHROPIC_API_KEY="sk-fake-key"
```

### 3. Avvia l'agente

Infine, lancia il programma specificando di usare un modello compatibile (in questo caso diciamo a LiteLLM di usare semplicemente il modello predefinito o passiamo un nome che LiteLLM mappa internamente):

```bash
claude
```
Ora tutte le tue richieste da terminale verranno processate in locale dal tuo Mac o PC!

## Domande frequenti

### È legale usare Claude Code senza API Key?
Sì. Stai semplicemente usando l'interfaccia CLI open-source (o scaricata localmente) e reindirizzando il traffico verso un tuo server privato. Non stai violando i sistemi originali, semplicemente non li stai chiamando.

### Qual è il miglior modello Ollama per il coding?
Attualmente, `qwen2.5-coder:32b` e le sue varianti offrono il miglior rapporto tra dimensione, capacità logiche e precisione nell'uso degli strumenti richiesti dagli agenti autonomi.

### Posso usare Claude Code completamente offline?
Assolutamente. Una volta scaricato il modello su Ollama e avviato LiteLLM in locale, l'intero flusso avviene sulla tua macchina (localhost). Puoi letteralmente spegnere il Wi-Fi e continuare a sviluppare assistito dall'AI.
