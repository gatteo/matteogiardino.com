Lookspan is the open-source platform released in 2026 that allows you to monitor and trace the LLM calls of your AI agents entirely locally. Running AI agents locally is great for privacy, but tracking logs and token usage quickly becomes a nightmare without the right tools. Lookspan solves exactly this problem: capable of processing over 1,000 spans per second with minimal performance impact, it’s an observability platform built for LLM workloads that you can run entirely on your own machine. After testing various expensive cloud SaaS solutions like LangSmith (which often exceed $200 per month for medium-scale loads), I decided to integrate Lookspan into my local stack. In this article, I'll show you how to set it up alongside OpenClaw and Ollama (if you're new to this stack, check out my [guide on using Ollama with OpenClaw](/blog/usare-ollama-con-openclaw-it)).

## What is Lookspan and why do AI agents need it?

Lookspan is an observability tool for AI agents that collects, visualizes, and analyzes traces of language model calls. Unlike traditional SaaS solutions that require you to send your prompt data to external servers, Lookspan is designed to be *local-first*.

When building autonomous agents, you execute dozens of LLM calls in the background: generation, validation, JSON parsing, and retries. If something breaks or token consumption spikes, debugging via console logs is practically impossible. Lookspan intercepts these flows, providing a clean dashboard with the duration of each span, the exact input, the generated output, and the token count.

## How to set up Lookspan locally

Installing Lookspan is quick if you use Docker. The platform consists of a trace collection server (typically compatible with OpenTelemetry) and a web UI.

### Step 1: Start Lookspan with Docker

The cleanest method to run Lookspan is via Docker Compose. Create a `docker-compose.yml` file with the official configuration:

```yaml
version: '3'
services:
  lookspan:
    image: lookspan/lookspan:latest
    ports:
      - "8080:8080" # Web UI
      - "4317:4317" # OTLP gRPC
      - "4318:4318" # OTLP HTTP
```

Then launch the container:

```bash
docker-compose up -d
```

Once started, the dashboard will be available at `http://localhost:8080`.

### Step 2: Integration with OpenClaw

If you use OpenClaw to manage your agents, you can easily configure it to send data to the Lookspan node you just created. Since it's compatible with the OpenTelemetry standard, you just need to set the correct environment variables before launching your agent:

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4318"
export OTEL_TRACES_EXPORTER="otlp"
export OTEL_METRICS_EXPORTER="none"
```

Now, every time your OpenClaw agent makes a call to Ollama or other local providers, the complete trace will be visible in the Lookspan dashboard.

## What went wrong the first time

One of the main issues I encountered involved port conflicts with other local stacks. Initially, I had an active Jaeger instance on port 4317. When I tried to launch Lookspan, the traces were dropped. The solution was simple: stop the old tracing containers and map Lookspan exclusively on OTLP, or change the port bindings in the `docker-compose.yml` and update the `OTEL_EXPORTER_OTLP_ENDPOINT`.

Also, remember that agent logs can grow rapidly. If you use Lookspan in a continuous testing environment, set a data retention policy to avoid filling up the disk on your Mac Mini or server.

## FAQ

### What is Lookspan?
Lookspan is an open-source platform for AI agent observability, specialized in tracing, logging, and monitoring LLM calls in local environments.

### What are the alternatives to Lookspan?
The most well-known cloud alternatives are LangSmith and Phoenix, while the open-source world often uses combinations of standard OpenTelemetry with Jaeger or SigNoz, though they are less specific for LLM prompts.

### Can I use Lookspan with Ollama?
Yes. Lookspan acts at the orchestrator level (e.g., OpenClaw or LangChain) intercepting direct API calls to Ollama, allowing you to monitor the input, output, and latency of your locally run models.

Observability is the missing link to move from simple AI scripts to truly reliable autonomous agents. Lookspan proves that you don't need to depend on expensive SaaS solutions or sacrifice data privacy to have professional dashboards and advanced tracing.
