Default system prompts often fail when using smaller local models. In this guide, I share the exact templates I use to eliminate hallucinations and make OpenClaw agents reliable.

When I started experimenting with OpenClaw and Ollama, the biggest issue wasn't hardware limits, but agent reliability. Smaller local models tend to get stuck in infinite tool loops or hallucinate capabilities they don't have.

## What is the OpenClaw System Prompt?

The system prompt is the operational "brain" of your agent. In OpenClaw, this prompt defines the rules of engagement: which tools to use, how to handle errors, and how to format outputs. Local models like Qwen or Llama 3 need stricter directives than massive models like GPT-4.

## How to Override the Default Prompt

To change your agent's core behavior, you need to modify your [OpenClaw configuration](/blog/setup-openclaw-without-paid-api-key-free). You can do this directly in your `openclaw.json` file.

```json
{
  "system_prompt": "You are a technical assistant. Use the tools provided. If a tool fails, stop execution and ask for user confirmation."
}
```

This approach prevents runaway executions.

## The "Zero-Hallucination" Template for Local Models

After dozens of tests, I've developed a template that stabilizes models under 14B parameters:

1. **Strict Role Definition**: No vague instructions.
2. **Forced JSON Format**: Force the model to use a structured thought/action format.
3. **The "I Don't Know" Rule**: If a command fails, the model must stop instead of trying random variations.

## Role-Based Agents: Coding and OSINT

Depending on the task, your prompt should adapt. A coding agent needs strict instructions on file handling, while an OSINT agent needs guidelines on summarizing search results without omitting key URLs.

## Common Prompting Mistakes

The most frequent mistake is overloading the prompt with too many rules. Local models suffer from "lost in the middle" syndrome. Keep the prompt concise, use bullet points, and place the most critical rules at the very end.
