I recently started evaluating open-source models with the Humanity's Last Exam (HLE) benchmark. The "blocking badgerbear" question is a logic puzzle within HLE designed to test frontier AI reasoning, spatial awareness, and constraint satisfaction. It is notoriously difficult for models under 70B parameters, making it an excellent stress test for local agents running via Ollama and OpenClaw.

## What is Humanity's Last Exam (HLE) Benchmark?

Humanity's Last Exam (HLE) is a rigorous benchmark meant to surpass the limitations of MMLU and GSM8K by testing multi-step logic and abstract reasoning. Instead of simple fact retrieval, HLE forces the model to construct internal representations of complex scenarios.

### Beyond MMLU and GSM8K

Older benchmarks have become saturated. Models routinely score above 90% on MMLU simply due to data contamination in their training sets. HLE introduces novel, non-searchable logic puzzles—like the badgerbear scenario—that require genuine deductive reasoning.

## The "Blocking Badgerbear" Question Explained

The "blocking badgerbear" test asks the model to navigate a hypothetical grid while avoiding a dynamically moving obstacle (the badgerbear). The model must output a valid sequence of moves that satisfies multiple overlapping constraints.

### Why models fail at this specific logic puzzle

Smaller models struggle because they lose track of the badgerbear's position after three or four steps. They attempt to predict the next token based on surface-level patterns rather than simulating the grid state in context memory.

## How to Run the HLE Benchmark Locally with OpenClaw

You can run the HLE badgerbear test directly on your local hardware using the lm-eval harness integrated with OpenClaw.

### Setting up the harness

Install the dependencies and run the specific task against your local Ollama endpoint:

```bash
pip install lm-eval
lm_eval --model local-chat-completions \
  --model_args base_url=http://localhost:11434/v1,model=llama3 \
  --tasks hle_badgerbear \
  --num_fewshot 0
```

## Results: Which Local Models Pass the Badgerbear Test?

I tested several models running locally on my Mac Studio. The results highlight a clear capability gap between small fast models and heavier reasoning engines.

### Qwen 3.5 vs Llama 3 vs Gemini 3.1 Pro (via API)

- **Llama 3 8B**: Fails completely. It hallucinates invalid moves by step 3.
- **Qwen 3.5 32B**: Achieves a 45% success rate, occasionally losing the spatial constraints.
- **Gemini 3.1 Pro (via API)**: Passes flawlessly 100% of the time, demonstrating true state-tracking.

If you are building autonomous agents, testing them on the badgerbear query provides immediate insight into their multi-step reasoning reliability.

## FAQ

### What is the blocking badgerbear in Humanity's Last Exam?
The blocking badgerbear is a specific logic puzzle in the HLE benchmark that tests an AI's spatial reasoning and constraint tracking over a sequence of steps.

### Can local LLMs pass the HLE benchmark?
Most models under 30B parameters struggle significantly with HLE. Larger models like Qwen 3.5 72B or API-based models like Gemini 3.1 Pro are required for reliable passing scores.

### How do I test my AI on the badgerbear puzzle?
You can use the `lm-eval` harness and point it to your local Ollama instance running the model, specifying the `hle_badgerbear` task.
