## Prerequisiti e Configurazione Locale

Prima di immergerti nella configurazione, devi installare alcuni strumenti essenziali per il funzionamento corretto del demone.

Installa e autentica la CLI di GitHub sul tuo sistema operativo:

```bash
brew install gh
gh auth login
```

Assicurati che l'utente che esegue il demone OpenClaw abbia i permessi corretti sul filesystem per eseguire `gh` all'interno della directory del tuo progetto. L'autoreview con openclaw: automatizza le pull request su github solo se i permessi sono corretti fin dall'inizio.

## Configurare l'Agente OpenClaw Autoreview

OpenClaw è dotato di una potente architettura a "Skill". Per abilitare le recensioni, l'agente si affida alle sue capacità native per `github`. Esso mappa gli output della CLI in un formato strutturato che l'LLM può eseguire in sicurezza.

Ecco il comando esatto che uso per attivare una openclaw autoreview su una PR specifica (in questo caso la 42):

```bash
openclaw exec -- "Use the GitHub CLI to download the diff of PR #42. Perform an openclaw autoreview: analyze the code for security vulnerabilities, memory leaks, and performance issues. If the code is solid, use 'gh pr review --approve'. If you find issues, use 'gh pr review --request-changes' and comment in detail."
```

### Il Processo in Background

Quando esegui quel comando, il motore esegue una sequenza chiara e prevedibile:
1. Esegue `gh pr diff 42` per leggere le righe modificate.
2. Elabora il diff attraverso l'LLM configurato.
3. Invia la revisione formattata a GitHub tramite `gh pr review`. 

In questo modo l'autoreview con openclaw: automatizza le pull request su github perfettamente e senza intoppi.

## Autoreview Avanzata: Esecuzione Pianificata

Eseguire manualmente è fantastico. Tuttavia l'automazione tramite cron è ancora meglio. Ho creato uno script bash che viene eseguito tramite Cron ogni mattina alle 8:00. Esso scansiona continuamente le PR non revisionate in coda.

```bash
#!/bin/bash
# openclaw-autoreview-cron.sh
cd /path/to/my/project

openclaw exec -- "Search for all open PRs. For each PR that lacks a review, perform an openclaw autoreview on the diff and post a constructive comment on the changes."
```

Questo flusso di lavoro è fantastico e ti salva ore di lavoro prezioso. Quando apro GitHub, ho già un'analisi iniziale del codice pronta. ## FAQ

### Quali modelli funzionano meglio per openclaw autoreview?
Consiglio di usare modelli specifici per il coding. Localmente, Qwen 2.5 Coder 32B o DeepSeek Coder sono scelte eccellenti. Se preferisci modelli in cloud, Claude 3.5 Sonnet fornisce un feedback incredibilmente dettagliato.

### L'openclaw autoreview sostituisce gli sviluppatori umani?
Assolutamente no. L'obiettivo è agire come un "developer junior instancabile" che gestisce i controlli ripetitivi (linting, sicurezza di base, linee guida di stile), liberando lo sviluppatore senior per concentrarsi sull'architettura e sulla logica di business.

### Può essere eseguito automaticamente ad ogni push?
Sì. Mentre il mio setup utilizza un job Cron giornaliero, puoi attivare il processo da un Git hook o da un leggero listener webhook locale collegato alle GitHub Actions.
