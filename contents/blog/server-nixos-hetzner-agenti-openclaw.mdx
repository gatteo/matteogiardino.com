Mantenere agenti AI attivi 24/7 è cruciale. La soluzione è imparare come installare openclaw su hetzner con nixos in 15 minuti. Questo ti permette di configurare un server Hetzner dedicato con NixOS per eseguire agenti AI in isolamento perfetto.

## Perché scegliere Hetzner e NixOS?

Quando esegui agenti AI autonomi, la sicurezza e la riproducibilità sono essenziali. Hetzner Metal offre una piattaforma incredibilmente economica ma con prestazioni pure, esattamente ciò che richiedono i carichi di lavoro intensivi degli agenti. NixOS completa perfettamente questo quadro. La sua configurazione dichiarativa significa che lo stato del server è perfettamente riproducibile. Se un agente fa danni o lo stato del sistema si corrompe, una semplice ricompilazione di Nix riporta tutto a una base funzionante.

## Passo 1: Scegliere il Server Hetzner

Per prima cosa, ti serve l'hardware giusto. Per i carichi di lavoro di OpenClaw, consiglio un server della linea AX di Hetzner. Queste macchine offrono eccellenti prestazioni single-core (vitali per i loop degli agenti) e molta RAM.

1. Accedi alla console Hetzner Robot.
2. Seleziona un server AX (ad esempio, AX41 o AX51).
3. Scegli il sistema Linux rescue predefinito  -  non installare ancora un sistema operativo standard.
4. Richiedi il server e attendi le credenziali di root.

## Passo 2: Installare NixOS in Rescue Mode

Una volta ottenuto l'accesso SSH al sistema rescue di Hetzner, installeremo NixOS utilizzando la configurazione dichiarativa. È qui che avviene la magia.

Per prima cosa, scarica l'immagine di installazione di NixOS e montala:

```bash
curl -L https://github.com/nix-community/nixos-images/releases/download/latest/nixos-kexec-installer-x86_64-linux.tar.gz | tar -xz
./kexec/run
```

La tua connessione SSH cadrà mentre il server si riavvia nell'installer di NixOS. Riconnettiti usando lo stesso IP e ti troverai nell'ambiente live di NixOS. Genera la configurazione hardware e modifica il tuo `configuration.nix` per includere le chiavi SSH e le impostazioni di rete di base.

## Passo 3: Mettere in Sicurezza l'Ambiente OpenClaw

Eseguire agenti con esecuzione `--yolo` su un server bare-metal è la ricetta per un disastro. Dobbiamo sfruttare l'isolamento nativo di NixOS.

Nel tuo `configuration.nix`, imposta un utente dedicato e un container systemd-nspawn per OpenClaw. Questo garantisce che l'agente non possa accedere al filesystem dell'host o ad altri servizi sensibili.

```nix
containers.openclaw = {
  autoStart = true;
  privateNetwork = true;
  config = { config, pkgs, ... }: {
    environment.systemPackages = [ pkgs.nodejs pkgs.git ];
    # Aggiungi qui la configurazione specifica di OpenClaw
  };
};
```

Questa sandbox dichiarativa garantisce che, anche se un agente allucina un comando distruttivo, il danno rimanga contenuto.

## Passo 4: Eseguire il Tuo Primo Agente

Con il container in esecuzione, puoi distribuire il tuo primo agente OpenClaw. Entra via SSH nel container e imposta l'ambiente:

```bash
nixos-container root-login openclaw
git clone https://github.com/your-repo/agent-config.git
cd agent-config
openclaw start --agent default
```

Ora il tuo agente è in esecuzione in un loop autonomo 24/7, isolato in modo sicuro dall'host. Puoi lasciarlo lavorare per processare task, monitorare API o gestire i tuoi flussi di lavoro.

## FAQ

### Hetzner è affidabile per carichi di lavoro AI?
Sì. Hetzner offre un ottimo rapporto prezzo-prestazioni. Un server AX41 è ideale per modelli locali e LLM, come si evince dalle ultime analisi del 2026.

### Posso eseguire Ollama sullo stesso server NixOS?
Assolutamente. Puoi dichiarare un servizio Ollama nella configurazione NixOS. [Uso di Ollama con OpenClaw](/blog/using-ollama-with-openclaw-it).

### Perché usare systemd-nspawn al posto di Docker?
NixOS si integra nativamente con systemd-nspawn, garantendo isolamento superiore.

## Prossimi Passi per la Tua Flotta di Agenti

Distribuire un solo agente è solo l'inizio. Con NixOS, scalare a più agenti è semplice: basta definire più container nel file di configurazione. Puoi anche integrare strumenti di memoria e storage persistente in modo sicuro all'interno del framework dichiarativo.

Combinando le prestazioni brute di Hetzner con la solida riproducibilità di NixOS, ottieni un ambiente OpenClaw pronto per la produzione a una frazione del costo di un servizio cloud gestito.
