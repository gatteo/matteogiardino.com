If you're trying to connect the Stitch MCP to Antigravity and hitting an API key error, you're not alone. The issue usually stems from how Antigravity handles environment variables for Model Context Protocol (MCP) servers compared to standard OpenClaw. Here is how to fix the "API key missing or invalid" error and get your agent working in minutes.

## Why the Stitch MCP API Key Error Happens

Antigravity IDE isolates the environment of its MCP servers to prevent token leakage. If you simply export your `STITCH_API_KEY` in your `.bashrc` or `.zshrc`, the Antigravity daemon won't see it when it spawns the Stitch integration. 

Unlike the standard OpenClaw CLI, which inherits your shell environment, Antigravity requires explicit configuration in its internal MCP settings file. You can learn more about this in our [OpenClaw common errors guide](/en/blog/debugging-openclaw-common-errors).

## Step 1: Locating the Antigravity MCP Config

First, make sure Antigravity is fully closed. Then, open your terminal and navigate to the Antigravity configuration directory.

On macOS and Linux:
```bash
nano ~/.antigravity/mcp-servers.json
```

On Windows:
```bash
notepad %USERPROFILE%\.antigravity\mcp-servers.json
```

If the file doesn't exist, create it. This is where Antigravity registers external tools like Stitch.

## Step 2: Adding the API Key to the Env Block

Find the entry for `stitch-mcp`. It should look something like this. You need to add the `env` block and place your API key inside it:

```json
{
  "mcpServers": {
    "stitch": {
      "command": "npx",
      "args": ["-y", "@stitch/mcp-server"],
      "env": {
        "STITCH_API_KEY": "sk-your-api-key-here"
      }
    }
  }
}
```

Make sure you replace `"sk-your-api-key-here"` with your actual Stitch API key. Do not leave trailing commas, as Antigravity requires strict JSON formatting.

## Step 3: Verifying the Connection

Restart Antigravity. If you are curious about how MCP works under the hood, check out our guide on [building a custom MCP server](/en/blog/building-custom-mcp-server-openclaw). Open the IDE and navigate to the **Agents** panel. Start a new chat and ask the agent:

*"Can you list the available tools from the Stitch MCP?"*

If the API key is configured correctly, the agent will respond with a list of available Stitch tools (like `stitch_create_project` or `stitch_sync`). If it says "MCP server disconnected," double-check your JSON syntax.

## Common Pitfalls

- **Missing Quotes:** JSON requires double quotes around both keys and values.
- **Wrong Key Name:** Ensure you use `STITCH_API_KEY` and not `STITCH_KEY` or `API_KEY`.
- **Daemon Restart:** Simply closing the Antigravity UI on macOS doesn't kill the background daemon. Use `pkill -f antigravity-daemon` before restarting the app to ensure it reads the new config.

## FAQ

**Where can I get a Stitch API Key?**
You can generate a new Stitch API key from your Stitch developer dashboard under "API Credentials". Remember to keep it secure and never share it in public repositories.

**Does Antigravity support other MCP servers?**
Yes. Antigravity IDE uses the standard Model Context Protocol, meaning any MCP server (like Playwright, GitHub, or Postgres MCPs) can be added to the `mcp-servers.json` file following the same structure.

**Why does Antigravity isolate environments?**
To ensure maximum security and prevent token leakage. By forcing explicit environment declarations in `mcp-servers.json`, Antigravity ensures that only approved tools get access to specific credentials.
