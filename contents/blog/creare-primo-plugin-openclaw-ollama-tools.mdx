### Prerequisiti per Scrivere Plugin per OpenClaw

Prima di iniziare a scrivere plugin per openclaw, assicurati di avere il seguente ambiente configurato:

## Passaggio 1: Inizializzare il Progetto

Per scrivere plugin per openclaw, devi prima preparare l'infrastruttura di base del tuo pacchetto npm. Apri il terminale e digita:

```bash
mkdir openclaw-ollama-plugin
cd openclaw-ollama-plugin
npm init -y
npm install @openclaw/sdk
```

Questo imposta un progetto Node.js pulito. La mia esperienza come sviluppatore mi ha insegnato che mantenere le dipendenze al minimo è fondamentale per evitare conflitti quando l'agente viene caricato in memoria.

## Passaggio 2: Scrivere la Logica del Plugin

Ora arriva la parte divertente: scrivere plugin per openclaw veri e propri. Crea un file chiamato `index.js` e inserisci questo codice:

```javascript
import { Plugin } from '@openclaw/sdk';

export class OllamaToolPlugin extends Plugin {
  constructor() {
    super({
      name: 'ollama-tool',
      description: 'Un tool per integrare Ollama localmente',
      version: '1.0.0'
    });
  }

  async execute(input) {
    console.log("Esecuzione del tool Ollama con input:", input);
    // Chiamata locale all'istanza Ollama sulla porta 11434
    const response = await fetch('http://localhost:11434/api/generate', {
      method: 'POST',
      body: JSON.stringify({
        model: 'llama3',
        prompt: input
      })
    });
    return response.json();
  }
}
```

Questa è l'architettura centrale che userai sempre quando devi scrivere plugin per openclaw. Intercetta la richiesta dell'agente e la inoltra direttamente all'istanza locale di Llama 3 tramite le API REST di Ollama.

## Passaggio 3: Registrazione dei Tool

Non basta solo scrivere plugin per openclaw; devi anche dichiarare all'agente quali strumenti hai messo a disposizione. Aggiungi il metodo `getTools`:

```javascript
  getTools() {
    return [
      {
        name: 'local_inference',
        description: 'Esegue inferenza locale tramite Ollama',
        parameters: {
          type: 'object',
          properties: {
            prompt: { type: 'string' }
          }
        }
      }
    ];
  }
```

In questo modo, l'orchestratore OpenClaw sa esattamente quali parametri inviare e quando utilizzare il tuo tool rispetto alle sue capacità native.

## Passaggio 4: Testare il Funzionamento

Una volta finito di scrivere plugin per openclaw, è il momento di provarlo sul campo. 

Avvia il tuo agente OpenClaw passando il flag del plugin:

```bash
openclaw run --plugin ./openclaw-ollama-plugin
```

Se tutto è stato configurato correttamente, l'agente ti risponderà utilizzando l'istanza locale di Ollama. Nessun dato lascerà il tuo computer.

### Gestione degli Errori nel Tuo Plugin

Mentre continui a scrivere plugin per openclaw, considera la gestione degli errori. Cosa succede se il demone Ollama non è in esecuzione?

```javascript
  async execute(input) {
    try {
      const response = await fetch('http://localhost:11434/api/generate', {
        // ... config
      });
      if (!response.ok) throw new Error("Ollama non risponde");
      return response.json();
    } catch (e) {
      console.error("Errore del plugin:", e.message);
      return { error: "Inferenza locale fallita. Assicurati che Ollama sia attivo." };
    }
  }
```

Una gestione robusta degli errori assicura che l'orchestratore OpenClaw possa ripiegare con grazia o avvisare l'utente invece di mandare in crash l'intero processo dell'agente.
