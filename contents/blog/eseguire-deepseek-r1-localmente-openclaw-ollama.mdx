## Perché DeepSeek R1 cambia le regole del gioco per l'AI locale

DeepSeek R1 porta un ragionamento di altissimo livello negli ambienti locali. A differenza dei modelli più piccoli che faticano con la logica multi-step, R1 eccelle nello scomporre problemi complessi. Lo sto usando da settimane nel mio setup locale, e la differenza nella generazione di codice e nel problem-solving è enorme.

Eseguendolo localmente, garantisci l'assoluta privacy dei dati e zero costi ricorrenti - l'ideale per elaborare dati aziendali sensibili o codebase interne.

## Prerequisiti: Installare Ollama e OpenClaw

Prima di iniziare, hai bisogno di due software:

1. **Ollama**: Il modo più semplice per eseguire LLM in locale. Vai su [ollama.com](https://ollama.com) e installalo per il tuo sistema operativo.
2. **OpenClaw**: Il framework agentico. Se non lo hai ancora, installalo semplicemente tramite npm:

```bash
npm install -g openclaw
```bash

## Scaricare ed eseguire DeepSeek R1 con Ollama

Una volta che Ollama è installato e in esecuzione, apri il tuo terminale. Dobbiamo scaricare i pesi del modello DeepSeek R1. A seconda della tua VRAM, potresti voler scegliere una versione quantizzata. Per la maggior parte dei MacBook o GPU moderne, i modelli standard 8B o 14B funzionano magnificamente.

Esegui questo comando:

```bash
ollama run deepseek-r1
```bash

Ollama scaricherà il modello e avvierà una sessione interattiva. Puoi digitare un prompt per testarlo, o semplicemente uscire (il modello rimarrà disponibile in background affinché OpenClaw possa connettersi).

## Configurare OpenClaw per usare il modello locale

Ora dobbiamo dire a OpenClaw di usare il nostro modello DeepSeek locale invece di un provider cloud. OpenClaw rende questo processo incredibilmente diretto.

Crea o modifica il tuo file di configurazione `openclaw.json` (di solito si trova in `~/.openclaw/openclaw.json`):

```json
{
  "models": {
    "primary": "ollama/deepseek-r1"
  }
}
```bash

Poiché OpenClaw comprende nativamente gli endpoint API di Ollama, non hai bisogno di configurare complessi URL di base o chiavi API. Instraderà automaticamente le richieste a `localhost:11434`.

## Caso d'uso reale: Un agente programmatore

Mettiamo alla prova questo setup creando un semplice agente programmatore autonomo. 

Crea un file chiamato `task.md` con le tue istruzioni:

```markdown
# Task
Scrivi uno script Python che monitora una directory locale per nuovi file immagine, li comprime e li sposta in una cartella di archivio.
```bash

Ora, esegui OpenClaw puntando a questo task:

```bash
openclaw run --task task.md --verbose
```bash

Osserva l'output del terminale. Vedrai il processo di ragionamento di DeepSeek R1 in azione: analizzerà i requisiti, scriverà lo script Python e sfrutterà gli strumenti di OpenClaw per salvare il file sul tuo disco.
