Faccio girare sei agenti su un Mac Mini connesso direttamente al mio telefono. OpenClaw Gateway è un livello di routing che collega gli agenti AI locali a interfacce di messaggistica esterne, e configurare ClawdBot e MoltBot su Telegram è il modo più veloce per chattare con loro. Dopo aver testato questa configurazione per 30 giorni, ecco esattamente come implementare l'integrazione.

## Cosa sono ClawdBot e MoltBot?

ClawdBot e MoltBot sono interfacce di integrazione per Telegram progettate per l'ecosistema OpenClaw. ClawdBot gestisce le chat generali con gli LLM e le chiamate agli strumenti, mentre MoltBot si concentra sulla gestione dei task in background degli agenti e sulle notifiche.

Collegandoli a OpenClaw Gateway, puoi chattare con i tuoi modelli locali (come Llama 3 che gira su [Ollama](/blog/qwen35-08b-openclaw-ollama-locale)) direttamente da Telegram, aggirando le dipendenze dal cloud.

## Configurare OpenClaw Gateway

Per prima cosa, devi avere il gateway in esecuzione in locale. Questo agisce come ponte tra i tuoi bot Telegram e il runtime locale di OpenClaw.

Inizia installando il pacchetto del gateway e generando una configurazione sicura:

```bash
npm install -g @openclaw/gateway
openclaw-gateway init --port 8080
```

Questo comando crea un file `gateway.json` nella tua directory `~/.openclaw/`.

## Collegare i Bot

Per connettere Telegram, ti servono i token dei bot forniti da BotFather.

### 1. Ottenere i token di Telegram

Apri Telegram, avvia una chat con `@BotFather` e usa il comando `/newbot` due volte per creare due bot. Salva i token API HTTP.

### 2. Configurare il gateway

Modifica il file `gateway.json` inserendo i token. Devi specificare l'endpoint e gli ID utente autorizzati, in modo che persone a caso non possano chattare con i tuoi agenti locali.

```json
{
  "telegram": {
    "clawdbot": {
      "token": "IL_TUO_TOKEN_CLAWD",
      "allowed_users": [123456789]
    },
    "moltbot": {
      "token": "IL_TUO_TOKEN_MOLT",
      "allowed_users": [123456789]
    }
  }
}
```

Riavvia il servizio del gateway:

```bash
openclaw-gateway start
```

## Risoluzione dei problemi con i Webhook

Quando ho fatto il primo deploy, Telegram non riusciva a raggiungere il mio Mac Mini locale perché si trovava dietro un NAT. Ho perso 2 ore a fare debug dei messaggi non ricevuti.

La soluzione è semplice: devi esporre il gateway tramite un reverse proxy come ngrok o Cloudflare Tunnels, quindi registrare esplicitamente l'URL del webhook.

```bash
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" -d "url=https://il-tuo-tunnel.example.com/webhook/clawdbot"
```

## Domande frequenti

### Cosa succede se OpenClaw Gateway va offline?
Telegram metterà in coda i messaggi per un massimo di 24 ore. Quando il gateway tornerà online, ClawdBot elaborerà i messaggi arretrati.

### Più utenti possono accedere ai bot?
Sì. Puoi aggiungere più ID utente Telegram all'array `allowed_users` nel file `gateway.json`.

### Funziona con Ollama?
Sì. OpenClaw Gateway instrada i messaggi verso qualsiasi modello tu abbia configurato nella tua installazione principale di OpenClaw.
