Configurare NullClaw offre un approccio rinfrescante per gli sviluppatori che desiderano la potenza di OpenClaw senza tutto il carico aggiuntivo. Di recente ho testato vari framework AI e NullClaw si è distinto come una soluzione incredibilmente veloce e leggera per eseguire agenti on-device.

Se stai cercando un framework per agenti completamente ridotto all'osso e local-first, questa guida fa per te. Ti mostrerò cos'è NullClaw, perché potresti preferirlo e esattamente come installarlo ed eseguirlo sulla tua macchina.

## Cos'è NullClaw?

NullClaw è un fork open source e minimalista del popolare framework OpenClaw (se non sai cos'è, leggi il [confronto tra OpenClaw e Hermes Agent](/blog/openclaw-vs-hermes-agent-quale-scegliere)). È stato progettato fin dall'inizio per eliminare dipendenze esterne, complesse configurazioni del gateway e peso inutile. Concentrandosi strettamente sull'esecuzione core dell'agente, fornisce un ambiente molto più snello.

A differenza della suite completa OpenClaw, che include ampie dashboard e telemetria, NullClaw è creato per gli sviluppatori che vogliono solo avviare un agente da riga di comando e lasciarlo lavorare direttamente con modelli locali come Ollama.

## Perché scegliere NullClaw rispetto a OpenClaw?

Ti starai chiedendo perché passare da un framework ricco di funzionalità a uno minimalista. La risposta risiede nelle prestazioni e nell'isolamento. 

Con NullClaw, l'impronta di memoria è notevolmente ridotta, rendendolo ideale per hardware datato o single-board computer come il Raspberry Pi. Inoltre, poiché elimina tutta la telemetria esterna e i ping al gateway, garantisce una privacy assoluta per ambienti enterprise o codebase sensibili.

## Prerequisiti per l'installazione

Prima di tuffarci nella configurazione, assicurati che il tuo sistema abbia i seguenti requisiti:

- **Node.js (v20+)**: Necessario per eseguire il runtime di NullClaw.
- **Ollama**: Per servire i modelli linguistici locali.
- **Git**: Per clonare il repository.

Dovresti anche avere un buon modello scaricato tramite Ollama. Consiglio `llama3` o `mistral` per compiti di ragionamento generali.

## Configurazione passo-passo di NullClaw

Il processo di installazione per NullClaw è molto semplice. Apri il tuo terminale e clona il repository ufficiale:

```bash
git clone https://github.com/nullclaw-ai/nullclaw-core.git
cd nullclaw-core
```

Successivamente, installa le dipendenze richieste utilizzando npm o pnpm:

```bash
npm install
```

Poiché NullClaw non richiede un complesso file di configurazione `openclaw.json`, ti basta creare un semplice file `.env` per puntare alla tua istanza Ollama:

```bash
echo "OLLAMA_HOST=http://127.0.0.1:11434" > .env
```

## Eseguire il tuo primo NullAgent

Terminata la configurazione, sei pronto per testare il framework. NullClaw utilizza una CLI estremamente semplificata. Avvia il tuo primo agente eseguendo:

```bash
npm run agent -- --task "Analizza i file nella directory corrente e elenca tutti gli script Python."
```

L'agente si connetterà al tuo modello locale, ragionerà sul compito ed eseguirà i comandi shell necessari. Noterai immediatamente quanto sia più veloce il tempo di avvio rispetto ai pesanti framework tradizionali.

Se apprezzi la velocità, la semplicità e il controllo locale assoluto, vale sicuramente la pena aggiungere NullClaw al tuo toolkit AI.


## Domande Frequenti (FAQ)

**NullClaw è compatibile con i plugin di OpenClaw?**
Sì, NullClaw supporta nativamente i plugin MCP standard a partire dalla versione 1.2 rilasciata all'inizio del 2026.

**Funziona su Raspberry Pi?**
Assolutamente sì. Con un'impronta di memoria di soli 45MB, NullClaw è perfetto per single-board computer come il Raspberry Pi 5.

**Ho bisogno di una chiave API per NullClaw?**
No, NullClaw si connette direttamente alla tua istanza Ollama locale (es. `http://127.0.0.1:11434`), quindi non sono necessarie chiavi OpenAI o Anthropic.
