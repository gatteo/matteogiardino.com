I run six agents on a Mac Mini connected directly to my phone. OpenClaw Gateway is a routing layer that connects local AI agents to external messaging interfaces, and setting up ClawdBot and MoltBot on Telegram is the fastest way to chat with them. After testing this setup for 30 days, here is exactly how to configure the integration.

## What are ClawdBot and MoltBot?

ClawdBot and MoltBot are Telegram integration interfaces designed for the OpenClaw ecosystem. ClawdBot handles general LLM chat and tool-calling, while MoltBot focuses on managing background agent tasks and notifications.

By connecting them to the OpenClaw Gateway, you can chat with your local models (like Llama 3 running on [Ollama](/en/blog/ollama-free-local-ai-assistant)) directly from Telegram, bypassing cloud dependencies.

## Setting up the OpenClaw Gateway

First, you need the gateway running locally. It acts as the bridge between your Telegram bots and the local OpenClaw runtime. 

Start by installing the gateway package and generating a secure configuration:

```bash
npm install -g @openclaw/gateway
openclaw-gateway init --port 8080
```

This creates a `gateway.json` file in your `~/.openclaw/` directory.

## Connecting the Bots

To connect Telegram, you need bot tokens from the BotFather.

### 1. Get your Telegram tokens

Open Telegram, chat with `@BotFather`, and use the `/newbot` command twice to create two bots. Save the HTTP API tokens.

### 2. Configure the gateway

Edit your `gateway.json` to include the tokens. You must specify the endpoint and the authorized user IDs so that random people cannot talk to your local agents.

```json
{
  "telegram": {
    "clawdbot": {
      "token": "YOUR_CLAWD_TOKEN",
      "allowed_users": [123456789]
    },
    "moltbot": {
      "token": "YOUR_MOLT_TOKEN",
      "allowed_users": [123456789]
    }
  }
}
```

Restart the gateway service:

```bash
openclaw-gateway start
```

## Troubleshooting Webhooks

When I first deployed this, Telegram could not reach my local Mac Mini because it was behind a NAT. I spent 2 hours debugging missed messages. 

The fix is simple: you must expose the gateway via a reverse proxy like ngrok or Cloudflare Tunnels, then register the webhook URL explicitly.

```bash
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" -d "url=https://your-tunnel.example.com/webhook/clawdbot"
```

## FAQ

### What happens if the OpenClaw gateway goes offline?
Telegram will queue messages for up to 24 hours. When the gateway comes back online, ClawdBot will process the backlog.

### Can multiple users access the bots?
Yes. You can add multiple Telegram user IDs to the `allowed_users` array in the `gateway.json` file.

### Does this work with Ollama?
Yes. OpenClaw Gateway routes the messages to whatever models you have configured in your main OpenClaw setup.
