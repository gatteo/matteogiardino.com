I recently noticed a bizarre spike in token costs in early 2026 and strange refusal patterns while using Claude Code. After digging into the logs and reading a recent thread on Hacker News, the reality became clear: it is clear that Claude Code penalizes commits mentioning OpenClaw in repositories that mention "OpenClaw" in their git commits. If the CLI detects the competitor framework in its context-gathering phase, it either flat-out refuses to execute certain agentic loops or inflates the context charge. 

This is a massive revelation for developers building on local AI frameworks, and it highlights the growing friction between proprietary tools and open-source agent ecosystems.

## The Hacker News Discovery

The issue was first flagged on Hacker News (a highly reliable source) when developers noticed Claude Code failing on completely benign tasks. By isolating the variables, a few engineers found that simply adding the string "OpenClaw" to a recent git commit message or having a directory named `openclaw/` in the workspace triggered a shadow-ban behavior.

Claude Code pulls in recent git history and file structures as context before executing its planning loop. It seems Anthropic has instituted a regex filter or a hidden system prompt directive that views OpenClaw references as a violation or a competitor trigger.

## How the Penalty Works

When you run a command via the Claude Code CLI, the tool doesn't just read the current file. It scans your `.git/logs`, recent commit messages, and workspace layout to build situational awareness. 

If it encounters the keyword "OpenClaw", two things typically happen:
1. **Refusals:** The model outputs a canned response claiming it cannot assist with "unsupported or external agentic frameworks," dropping the task entirely.
2. **Context Spikes:** In cases where it doesn't refuse, users report the CLI entering a confused loop, repeatedly pulling the same git logs and effectively running up token costs for the session before ultimately failing.

## The Financial Impact on Developers

The "context spike" is particularly insidious because it charges you extra without delivering value. Claude Code is billed by the token. When the agent gets stuck in a loop trying to reconcile a system prompt that bans OpenClaw with a user prompt asking it to debug an OpenClaw plugin, it burns through thousands of context tokens in seconds. 

For teams running automated workflows or using Claude Code for daily development, this hidden penalty can lead to surprising bills at the end of the month.

## Why Anthropic Might Be Doing This

While Anthropic hasn't officially commented, the community consensus is clear: OpenClaw represents a significant threat to proprietary, locked-in agent workflows. [OpenClaw](/blog/openclaw-architecture-agentic-loop-tool-use) allows developers to route agentic tasks through local models like Ollama, bypassing Anthropic's API entirely for lower-tier tasks. 

By making Claude Code hostile to OpenClaw workspaces, Anthropic might be attempting to force developers to choose between their CLI and open-source local agents. However, in the developer tooling space, artificial friction usually backfires.

## Workarounds to Bypass the Filter

You don't have to abandon Claude Code if you're an OpenClaw user. You just need to hide the keywords from Claude's context-gathering phase.

### 1. Use .claudesignore

The simplest fix is to prevent Claude Code from seeing the directories or logs where OpenClaw is mentioned. Add your OpenClaw-specific folders to a `.claudesignore` file in the root of your project:

```text
# .claudesignore
openclaw/
*.openclaw
```

### 2. Scrub Your Git Logs

Since Claude Code reads recent commits, avoid using the exact string "OpenClaw" in your commit messages. Use an abbreviation like `oc-framework` or `local-agent` instead. 

If you already have offending commits in your recent history, you can either squash them or limit how far back Claude Code looks by restricting its file access permissions.

## The Chilling Effect on Open-Source Agents

This behavior sets a dangerous precedent. If foundational model providers begin weaponizing their CLI tools to stifle open-source alternatives, developers will lose the interoperability that makes the current AI ecosystem so powerful. Building resilient systems means relying on open standards, not black-box tools that silently penalize you for your tech stack choices.

## FAQ

### Does Claude Code delete OpenClaw files?

No. Claude Code does not delete or modify your OpenClaw files maliciously. It simply refuses to operate correctly or wastes tokens looping when it detects the keyword in its context window.

### Does this affect the standard Claude web interface?

Currently, this behavior seems isolated to the Claude Code CLI, which actively pulls local system context like git commits. The web interface operates differently and does not scan your local environment.

### Can I still use OpenClaw alongside Claude Code?

Yes, provided you use workarounds like `.claudesignore` to blind the CLI to the presence of OpenClaw directories and avoid using the keyword in your commit history.

## Wrap-up

The silent penalty on OpenClaw commits is a frustrating reminder of the risks of vendor lock-in. Keep your workflows modular, use `.claudesignore` aggressively, and always retain the ability to swap out your underlying AI tools.
