## Cosa è DreamX-World?

DreamX-World è un modello di mondo interattivo all'avanguardia che effettua generazione predittiva. A differenza dei classici LLM che elaborano solo testo o immagini statiche, questo modello simula ambienti completi e prevede risultati basati sulla fisica.

Integrato con gli agenti OpenClaw, permette alla tua IA di "vedere" le conseguenze prima di eseguire le azioni. È perfetto per task di ragionamento spaziale complesso, robotica e pipeline di automazione avanzate.

## Come configurare DreamX-World con OpenClaw

### Passo 1: Installare il modello in locale

Per prima cosa, devi scaricare i pesi del modello locale. Assicurati di avere almeno 16GB di VRAM per il modello base.

```bash
openclaw pull dreamx-world:latest
```

### Passo 2: Configurare il Server MCP

Collega il modello al tuo ambiente usando il server Model Context Protocol (MCP) in modo che gli agenti possano interrogare il motore fisico.

```bash
openclaw mcp start dreamx-world --port 8123
```

## Cosa è andato storto nei miei test

Il problema principale che ho riscontrato è stato l'overflow della VRAM. Il modello si bloccava ripetutamente gestendo scene con più di 5 entità in movimento. La soluzione è stata limitare la finestra di contesto nel file di configurazione di OpenClaw a `ctx_size: 4096` e abilitare il memory offloading.

## FAQ

### Cosa sono i modelli di mondo interattivi?
I modelli di mondo interattivi sono sistemi AI capaci di simulare e prevedere il comportamento fisico degli ambienti in tempo reale, permettendo agli agenti di testare i risultati prima di agire.

### Posso far girare DreamX-World in locale?
Sì, puoi eseguirlo in locale, ma richiede una potenza di calcolo significativa. Si consiglia un minimo di 16GB di VRAM per prestazioni stabili insieme ai tuoi modelli agenti principali.

### DreamX-World funziona con Claude Code?
Sì, può essere integrato tramite il Model Context Protocol (MCP), permettendo a Claude Code e agli agenti OpenClaw di sfruttare le sue previsioni spaziali.
