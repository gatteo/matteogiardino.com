Eseguire Claude Code di Anthropic richiede normalmente una API key attiva, legandoti ai costi del cloud. Tuttavia, grazie a un recente aggiornamento di Ollama che introduce la compatibilità con le API Messages di Anthropic, ora è possibile "ingannare" Claude Code per fargli usare i tuoi modelli locali in modo completamente gratuito.

In questa guida, ti mostrerò come configurare Claude Code per interfacciarsi con un'istanza locale di Ollama (usando modelli come Qwen 2.5 Coder o Llama 3) senza spendere un centesimo. Per un confronto più ampio sulle interfacce degli agenti, dai un'occhiata alla mia analisi su [OpenClaw vs Claude Code](/blog/openclaw-vs-claude-code-confronto).

## L'Importanza delle Messages API

Fino al 2026, i modelli locali erano perlopiù isolati e Ollama serviva la generazione di testo tramite i suoi endpoint dedicati `/api/generate`. Adottando lo standard delle Messages API di Anthropic, Ollama ha creato un ponte tra gli strumenti cloud enterprise e i modelli open-weight.

Ciò significa che qualsiasi strumento costruito esclusivamente per la famiglia Claude di Anthropic può ora interfacciarsi con un modello locale come Llama o Qwen, senza richiedere agli sviluppatori di scrivere adattatori personalizzati. Questo livello API standardizzato sta democratizzando l'accesso agli strumenti di sviluppo più avanzati.
## Prerequisiti per Claude Code Locale

Per far funzionare il tutto, hai bisogno di hardware in grado di far girare modelli da programmazione in locale. Nel mio setup uso un Mac Mini M4 Pro, ma qualsiasi sistema con abbastanza VRAM per Ollama andrà bene.

Ecco cosa ti serve:

1. **Ollama (versione 0.1.42 o superiore)**: L'ingrediente segreto è il supporto alle Messages API di Anthropic. Versioni vecchie non funzioneranno.
2. **Un modello capace di fare coding**: Hai bisogno di modelli che supportino il function calling (tool use) e abbiano una context window generosa. Ti consiglio `qwen2.5-coder:7b` o `qwen2.5-coder:14b`.
3. **Claude Code**: La CLI ufficiale di Anthropic.

## Scaricare i Modelli Giusti

Assicurati di avere il modello adatto. Deve avere una context window di almeno 32k o 64k per gestire la codebase correttamente.

Esegui questo nel terminale:

```bash
ollama pull qwen2.5-coder:7b
```

Una volta scaricato, mantieni Ollama in esecuzione in background.

## Installare e Configurare Claude Code

Se non hai ancora installato Claude Code, puoi farlo tramite npm. Se invece hai già fatto login su Anthropic tramite Claude Code, devi prima fare il logout, poiché vogliamo bypassare l'autenticazione remota.

```bash
npm install -g @anthropic-ai/claude-code
claude logout
```

Ora arriva il trucco: useremo le variabili d'ambiente per far puntare Claude Code al server locale di Ollama invece che agli endpoint cloud di Anthropic.

```bash
export ANTHROPIC_API_KEY="***"
export ANTHROPIC_BASE_URL="http://localhost:11434"
```

La `ANTHROPIC_API_KEY` può essere qualsiasi stringa. Ollama non la valida, ma l'SDK si aspetta che la variabile esista. La `ANTHROPIC_BASE_URL` punta l'SDK alla porta locale di Ollama.

<Image src="/images/blog/claude-code-ollama-models-without-api-key/claude-code-with-ollama-models-without-anthropic-api-key-175.webp" alt="Impostare variabili d'ambiente" width={1200} height={630} />

## Usare Claude Code in Locale

Con le variabili d'ambiente configurate, puoi lanciare Claude Code e passargli il nome del tuo modello locale, proprio come se fosse un modello di Claude.

```bash
claude -m qwen2.5-coder:7b
```

Claude Code si avvierà mostrando l'interfaccia e i colori tipici di Anthropic, ma sotto il cofano ogni prompt e ogni lettura della codebase verrà instradata sul tuo computer locale.

Puoi chiedergli di generare codice, sistemare errori TypeScript o fare refactoring di componenti. Dato che Ollama gestisce perfettamente il tool-calling, Claude Code riuscirà a leggere i file ed eseguire i comandi altrettanto bene quanto la versione cloud.

## Domande Frequenti

**Funziona con qualsiasi modello Ollama?**  
No. Il modello deve supportare il function calling (i tool) in modo robusto. Modelli come Qwen 2.5 Coder o Llama 3.1 funzionano meglio. Modelli più datati andranno in errore quando Claude Code proverà a leggere i file.

**Perché non usare semplicemente OpenClaw?**  
OpenClaw è fantastico per workflow personalizzati e per l'orchestrazione. Tuttavia, se vuoi usare specificamente l'interfaccia CLI di Claude Code ma azzerando i costi API, questo trucco fa al caso tuo.

**È intelligente quanto Claude 3.5 Sonnet?**  
Nessun modello locale da 7B o 14B eguaglia attualmente le capacità logiche di Sonnet. Baratti la pura capacità di ragionamento con la privacy totale e zero costi API.
