I system prompt predefiniti spesso falliscono quando si utilizzano modelli locali più piccoli. In questa guida, ho raccolto i template che utilizzo per eliminare le allucinazioni e far lavorare gli agenti OpenClaw in modo affidabile.

Quando ho iniziato a sperimentare con [OpenClaw e Ollama](/blog/guida-qwen-35-ollama-openclaw), il problema più grande non era la potenza dell'hardware, ma l'affidabilità dell'agente. I modelli locali tendono a perdersi in loop infiniti o a inventare strumenti che non esistono.

## Cos'è il System Prompt in OpenClaw?

Il system prompt è il "cervello" operativo del tuo agente. In OpenClaw, questo prompt definisce le regole d'ingaggio: quali strumenti usare, come rispondere in caso di errore e come formattare l'output. I modelli locali come Qwen o Llama 3, per quanto potenti, necessitano di direttive più restrittive rispetto a un GPT-4.

## Come sovrascrivere il prompt predefinito

Per cambiare il comportamento base del tuo agente, devi modificare la configurazione di OpenClaw. Puoi farlo direttamente nel file `openclaw.json` o passandolo come parametro all'avvio.

```json
{
  "system_prompt": "Sei un assistente tecnico. Usa gli strumenti a tua disposizione. Se uno strumento fallisce, ferma l'esecuzione e chiedi conferma."
}
```

Questo approccio previene azioni distruttive incontrollate.

## Il Template "Zero-Allucinazioni" per Modelli Locali

Dopo decine di test, ho sviluppato un template che stabilizza i modelli sotto i 14B di parametri:

1. **Definizione del ruolo stretta**: Niente descrizioni vaghe.
2. **Formato JSON forzato**: Obbliga il modello a usare un formato di pensiero/azione strutturato.
3. **Regola del "Non so"**: Se un comando fallisce, il modello deve fermarsi, non tentare varianti casuali.

## Agenti Basati sul Ruolo: Coding e OSINT

A seconda del compito, il prompt deve cambiare. Un agente per la programmazione avrà bisogno di istruzioni severe sulla gestione dei file, mentre un agente OSINT avrà bisogno di linee guida su come riassumere i risultati di ricerca senza omettere URL chiave.

## Errori Comuni di Prompting

L'errore più frequente è sovraccaricare il prompt con troppe regole. I modelli locali soffrono di "lost in the middle", dove dimenticano le istruzioni poste a metà del testo. Mantieni il prompt conciso, usa elenchi puntati e inserisci le regole vitali alla fine.
