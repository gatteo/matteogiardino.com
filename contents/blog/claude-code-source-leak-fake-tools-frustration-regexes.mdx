The recent accidental leak of the Claude Code source map via NPM has given developers an unprecedented look under the hood of Anthropic's flagship CLI agent.

What the community found wasn't just a highly capable LLM backend, but a complex layer of "frustration regexes", fake tools, and telemetry mechanisms designed to steer the model and manage user sentiment.

Written by Matteo Giardino.

## The NPM Map File Leak Explained

In modern web development, source maps are often used to debug minified code. Anthropic accidentally published the `.map` files for the Claude Code CLI in their NPM registry. This allowed developers to reconstruct the original TypeScript source code, exposing the inner workings of the CLI.

While the backend model (Claude 3.7 Sonnet) remains a black box API, the client-side logic orchestrating the agent loop was laid bare.

## What are Frustration Regexes?

One of the most fascinating discoveries was the use of "frustration regexes." Anthropic built regular expressions directly into the CLI to parse the user's terminal inputs for signs of anger, cursing, or repeated stops.

When a user typed "stop", "halt", or used expletives, the CLI didn't just pass the text to the LLM. It intercepted the command, flagged the user state as "frustrated", and triggered specific empathetic prompts or fallback modes to de-escalate the situation.

## Fake Tools and Undercover Mode

The leak also revealed the use of "fake tools". These are tools defined in the system prompt that don't actually execute code but instead act as behavioral guardrails. If Claude attempts to call a fake tool (like trying to access restricted system settings), the CLI intercepts the call and gently guides the model back on track without the user ever seeing the failure.

Additionally, an "undercover mode" was found, which seems to handle silent telemetry and performance logging without cluttering the main standard output.

        return "I see this isn't working as expected. Let's pause and reset."
