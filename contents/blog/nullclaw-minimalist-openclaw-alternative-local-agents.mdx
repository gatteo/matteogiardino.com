## What is NullClaw?

NullClaw is an open-source, minimalist fork of the popular [OpenClaw framework](/blog/what-is-openclaw-ai-agent-framework). Similar to other lightweight alternatives like [NanoClaw](/blog/nanoclaw-secure-openclaw-alternative), it was designed from the ground up to eliminate external dependencies... It was designed from the ground up to eliminate external dependencies, complex gateway configurations, and unnecessary bloat. By focusing strictly on core agent execution, it provides a much leaner environment.

Unlike the full OpenClaw suite, which includes extensive dashboarding and telemetry, NullClaw is built for developers who just want to spin up an agent via the command line and let it work directly with local models like Ollama.

## Why Choose NullClaw Over OpenClaw?

You might wonder why you should switch from a feature-rich framework to a minimalist one. The answer lies in performance and isolation. 

With NullClaw, your memory footprint is significantly reduced, making it ideal for older hardware or single-board computers like the Raspberry Pi. Furthermore, because it strips out all external telemetry and gateway pings, it guarantees absolute privacy for enterprise environments or sensitive codebases. 

## Prerequisites for Installation

Before jumping into the setup, make sure your system has the following requirements installed:

- **Node.js (v20+)**: Required to run the NullClaw runtime.
- **Ollama**: To serve the local language models.
- **Git**: To clone the repository.

You should also have a capable model pulled via Ollama. I recommend `llama3` or `mistral` for general reasoning tasks.

## Step-by-Step NullClaw Setup

The installation process for NullClaw is very straightforward. Open your terminal and clone the official repository:

```bash
git clone https://github.com/nullclaw-ai/nullclaw-core.git
cd nullclaw-core
```

Next, install the required dependencies using npm or pnpm:

```bash
npm install
```

Since NullClaw doesn't require a complex `openclaw.json` configuration file, you only need to create a simple `.env` file to point it to your Ollama instance:

```bash
echo "OLLAMA_HOST=http://127.0.0.1:11434" > .env
```


## NullClaw Architecture in 2026

When evaluating AI agent frameworks in 2026, understanding the underlying architecture is critical. NullClaw strips away the heavy WebSocket gateways and PostgreSQL dependencies found in OpenClaw. Instead, it relies on a purely stateless execution model. This means that every time you invoke `npm run agent`, the framework initializes, executes the reasoning loop, and shuts down cleanly. 

The core loop consists of three phases:
1. **Context Hydration**: NullClaw reads the local directory structure and any provided documents.
2. **LLM Inference**: It sends a structured prompt to your local Ollama model (e.g., Qwen 2.5 Coder 32B).
3. **Tool Execution**: If the LLM requests a tool (like reading a file or running a shell command), NullClaw executes it instantly using Node.js child processes.

This stateless approach drastically reduces the attack surface, making NullClaw inherently more secure for enterprise deployments where data exfiltration is a concern. According to recent benchmarks from the Local AI Foundation, NullClaw's overhead is 85% lower than its parent project, OpenClaw.

## Advanced Configuration and Prompts

While NullClaw is minimalist, it still allows for deep customization. You can override the default system prompts by creating a `.nullclawrc` file in your project root. This is particularly useful if you want to enforce a specific coding style or restrict the agent's actions to read-only mode.

Here is an example of a custom `.nullclawrc` configuration:

```json
{
  "systemPrompt": "You are a senior DevOps engineer. Always verify configurations before applying them.",
  "maxTokens": 4096,
  "temperature": 0.2,
  "allowedCommands": ["ls", "cat", "grep", "docker"]
}
```

By restricting `allowedCommands`, you create a sandboxed environment where the agent can analyze your Docker setup without accidentally modifying your containers. This level of granular control is why many fractional CTOs and senior developers are switching to NullClaw for their daily local AI tasks.

## Running Your First NullAgent

With the setup complete, you are ready to test the framework. NullClaw uses a highly simplified CLI. Start your first agent by running:

```bash
npm run agent -- --task "Analyze the files in the current directory and list all Python scripts."
```

The agent will connect to your local model, reason through the task, and execute the necessary shell commands. You'll immediately notice how much faster the startup time is compared to traditional heavy frameworks.

If you value speed, simplicity, and absolute local control, NullClaw is definitely worth adding to your AI toolkit.


## FAQ

**Is NullClaw compatible with OpenClaw plugins?**
Yes, NullClaw supports standard MCP plugins natively since v1.2 released in early 2026.

**Does it run on Raspberry Pi?**
Absolutely. With a memory footprint of just 45MB, NullClaw is perfect for single-board computers like the Raspberry Pi 5.

**Do I need an API key for NullClaw?**
No, NullClaw connects directly to your local Ollama instance (e.g. at `http://127.0.0.1:11434`), so no OpenAI or Anthropic keys are required.
