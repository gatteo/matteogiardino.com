Aggiornare OpenClaw richiede un solo comando, ma sbagliare versione può bloccare i tuoi agenti. Io aggiorno il mio server OpenClaw su Mac Mini almeno una volta a settimana nel 2026. Dopo 3 mesi di utilizzo intenso con oltre 10 agenti in produzione, ho capito che il modo peggiore per aggiornare è reinstallare tutto da zero. 

In questa guida pratica vediamo come usare `openclaw update`, come forzare un downgrade quando un rilascio sperimentale rompe le dipendenze, e come gestire il riavvio del gateway. 

Se vuoi approfondire la struttura base del framework prima di procedere, leggi la mia guida su [cos'è OpenClaw e come funziona](/blog/cos-e-openclaw-framework-agenti-ai).

## Come Aggiornare OpenClaw in modo sicuro (Linux, Mac e WSL2)

Il modo più sicuro per aggiornare OpenClaw non è scaricare script bash a caso, ma usare il comando nativo. L'aggiornamento richiede circa 15 secondi per scaricare i binari.

```bash
openclaw update
```

Questo comando scarica l'ultima versione stabile e fa l'upgrade in place. Durante l'aggiornamento, il sistema sostituisce il core (circa 180 MB di binari) e la CLI, e riavvia automaticamente il Gateway.

<Image src="/images/blog/aggiornare-openclaw/how-to-update-openclaw-97.webp" alt="Comando openclaw update in esecuzione su Linux" width={1200} height={630} />

**Cosa succede durante l'aggiornamento:**

- Rileva l'installazione esistente.
- Scarica e installa l'ultima versione.
- Aggiorna core e CLI.
- Esegue il doctor per health check e integrità.
- Riavvia i servizi se necessario.

<Image src="/images/blog/aggiornare-openclaw/how-to-update-openclaw-78.webp" alt="Log di aggiornamento OpenClaw con verifica dei plugin" width={1200} height={630} />

Vedrai i log mentre aggiorna plugin e riavvia i servizi. Se non hai installato plugin extra, quella fase verrà saltata.

### Prompt Interattivi Durante l'Update

Durante l'aggiornamento potresti incontrare questi prompt:

**Tighten permissions to 700:** se sei in produzione, ti consiglio di scegliere yes per hardening della sicurezza.

**Upgrade gateway service config:** se hai integrato OpenClaw con [OpenAI su AWS EC2](/blog/installare-openclaw-aws-ec2) o con Anthropic, seleziona yes. Se hai installato con Ollama o LM Studio, puoi saltare e riavviare il servizio manualmente. In quel caso, potresti voler rileggere la guida su come [eseguire LLM in locale con OpenClaw e Ollama](/blog/qwen35-08b-openclaw-ollama-locale).

## Come fare il Downgrade di OpenClaw a una versione specifica

Se hai appena provato ad aggiornare OpenClaw e un tuo workflow si è rotto, puoi tornare indietro facilmente. Il comando di downgrade mi ha salvato letteralmente ore di lavoro quando un update ha rotto i miei webhook. 

Per fare il downgrade a una versione specifica, come ad esempio la versione `0.8.2`:

```bash
openclaw update --version 0.8.2
```

Se vuoi semplicemente tornare all'ultima versione stabile approvata (Stable Channel), ignorando eventuali tag sperimentali (i canali `--channel beta` o `--channel dev`):

```bash
openclaw update --channel stable
```

Se continui ad avere problemi con i comandi o gli script, puoi [debuggare OpenClaw e risolvere gli errori comuni](/blog/debuggare-openclaw-errori-comuni) usando gli strumenti integrati.

## Verifica dell'aggiornamento: il Riavvio del Gateway

Dopo un aggiornamento importante, assicurati che il gateway OpenClaw stia utilizzando i nuovi binari. L'operazione è immediata:

```bash
openclaw gateway restart
```

E per verificare che tutto stia girando senza errori bloccanti:

```bash
openclaw gateway status
```

Se il servizio LaunchAgent è bloccato su Mac (o systemd su Linux), il comando `status` ti mostrerà l'uptime in ms o l'errore esatto per il debug. Ne ho parlato in dettaglio nella mia guida su [come riavviare il gateway OpenClaw](/blog/riavviare-gateway-openclaw-verificare-stato).

<Image src="/images/blog/aggiornare-openclaw/how-to-update-openclaw-277.webp" alt="Status di tutti i servizi OpenClaw dopo aggiornamento" width={1200} height={630} />

Il comando `openclaw status` mostra tutti i servizi. Puoi ignorare warning sui privilegi di gruppo dove stanno rendendo il sistema più sicuro.

## Risoluzione dei problemi con OpenClaw Doctor

A volte un aggiornamento di OpenClaw o un downgrade lascia dei permessi non corretti nelle cartelle temporanee sotto `~/.openclaw`. Il modo più veloce per risolvere:

```bash
openclaw doctor
```

Questo strumento controlla la versione di Node, i permessi delle directory locali e i conflitti sulla porta di default del Gateway (solitamente la porta TCP `18789`).

## Altri Metodi di Aggiornamento

### Aggiornamento via npm Globale

Se hai installato OpenClaw come pacchetto npm globale:

```bash
npm install -g openclaw@latest
```

Questo metodo è semplice e diretto, perfetto se gestisci OpenClaw come una normale dipendenza Node. L'installazione di solito richiede 20-30 secondi su una connessione in fibra. Puoi leggere la documentazione ufficiale di NPM per maggiori dettagli all'indirizzo `https://www.npmjs.com/package/openclaw`.

<Image src="/images/blog/aggiornare-openclaw/how-to-update-openclaw-342.webp" alt="Aggiornamento OpenClaw con npm install globale" width={1200} height={630} />

### Aggiornamento Docker

Se usi Docker, il processo prevede l'arresto del container, il pull e il riavvio:

```bash
docker stop openclaw
docker pull openclaw/openclaw:latest
docker run -d --name openclaw openclaw/openclaw:latest
```

Fermi il container corrente, scarichi l'immagine più recente e riavvii. Scegli il metodo che si integra meglio con la tua pipeline CI/CD.

## Canali: Stable, Beta, Dev

OpenClaw ha tre canali di rilascio. Come per i browser, decidi tu quanto vuoi rischiare:

- **Stable:** consigliato per produzione o use case seri.
- **Beta:** nuove funzionalità in test, relativamente stabile.
- **Dev:** bleeding edge, per sperimentare con le ultime novità.

<Image src="/images/blog/aggiornare-openclaw/how-to-update-openclaw-420.webp" alt="Canali di aggiornamento OpenClaw: stable, beta, dev" width={1200} height={630} />

Puoi cambiare canale in qualsiasi momento con:

```bash
openclaw update --channel beta
```

Per vedere cosa verrebbe fatto senza applicare le modifiche:

```bash
openclaw update --dry-run
```

Questo è utile per testare upgrade potenzialmente rischiosi prima di applicarli sul server principale. ## Domande Frequenti su Aggiornamenti

### Posso saltare l'esecuzione del doctor?
Sì, ma è fortemente sconsigliato. Il doctor sistema permessi che, se ignorati, potrebbero impedire l'avvio del gateway al riavvio del server locale.

### L'aggiornamento cancella i log e la memoria?
No. Il comando `openclaw update` non tocca mai le cartelle della memoria né la cronologia delle chat. L'aggiornamento interviene solo sui binari del core e della CLI installati nel sistema.

### Cosa faccio se il downgrade fallisce?
Se `openclaw update --version 0.8.2` fallisce, la soluzione migliore è eseguire [openclaw doctor --fix per risolvere errori comuni](/blog/debuggare-openclaw-errori-comuni) per ripristinare i permessi e riprovare, oppure usare il downgrade via npm se avevi installato il framework globalmente.

## Riepilogo

**Per la maggior parte degli utenti:**
- Esegui `openclaw update`.
- Scegli yes ai prompt di sicurezza se sei in produzione.
- Riavvia il gateway service con `openclaw gateway restart`.
- Verifica con `openclaw status`.

OpenClaw è un progetto in rapida evoluzione. Tenerlo aggiornato significa avere accesso alle ultime funzionalità, fix di sicurezza e miglioramenti di performance. Con questa guida hai tutto ciò che ti serve per gestire upgrade e downgrade in sicurezza.
