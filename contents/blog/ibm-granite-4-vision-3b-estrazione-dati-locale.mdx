Di recente ho avuto la necessità di estrarre dati strutturati da centinaia di PDF aziendali senza poterli inviare alle API di OpenAI o Anthropic per questioni di privacy. **IBM Granite 4.0 Vision 3B** è un modello vision-language (VLM) ultracompatto e open source progettato specificamente per convertire documenti scansionati, tabelle e grafici in JSON e CSV direttamente in locale. Dopo averlo testato a fondo sui server del mio ufficio, vi mostro come installarlo e usarlo.

## Cos'è IBM Granite 4.0 Vision 3B

IBM Granite 4.0 3B Vision è un modello ibrido che unisce un LLM base (Granite 4.0 Micro da 3.5B) con un adattatore LoRA visuale da 0.5B parametri. Questa architettura permette di avere un modello leggerissimo (meno di 4 miliardi di parametri in totale) capace di comprendere layout documentali complessi con una precisione tipica di modelli molto più pesanti.

È stato addestrato pesantemente su ChartNet, un dataset da un milione di grafici e tabelle. Di conseguenza, eccelle dove i modelli più piccoli solitamente falliscono: la comprensione spaziale di ricevute, fatture e report finanziari.

## Estrazione Dati: Tabelle, Grafici e KVP

La vera forza di Granite Vision 3B è la sua formattazione dell'output. Può essere istruito per restituire i dati nei seguenti formati:

1. **Chart2CSV e Chart2Summary**: Estrae i valori numerici direttamente dalle barre e linee di un grafico scansionato.
2. **Table Extraction**: Trasforma tabelle complesse e nidificate in HTML, OTSL o JSON strutturato.
3. **KVP (Key-Value Pair)**: Estrae valori semantici basandosi sui nomi dei campi, indipendentemente da dove si trovino nel layout del documento.

Questo lo rende ideale per l'integrazione in pipeline automatizzate con agenti AI.

## Come Installare Granite 4.0 Vision in Locale

Ci sono diversi modi per avviare il modello. Il metodo più performante per l'inferenza è utilizzare **sglang** o Docker. 

### Avvio tramite Docker

Se hai una GPU e driver NVIDIA installati, puoi lanciare un server compatibile con le API di OpenAI in pochi secondi:

```bash
docker run --gpus all \
    --shm-size 32g \
    -p 30000:30000 \
    -v ~/.cache/huggingface:/root/.cache/huggingface \
    --ipc=host \
    lmsysorg/sglang:latest \
    python3 -m sglang.launch_server \
        --model-path "ibm-granite/granite-4.0-3b-vision" \
        --host 0.0.0.0 \
        --port 30000
```

### Effettuare una chiamata API

Una volta avviato il server, puoi inviargli immagini ed estrarre testo proprio come faresti con GPT-4 Vision:

```bash
curl -X POST "http://localhost:30000/v1/completions" \
  -H "Content-Type: application/json" \
  --data '{
    "model": "ibm-granite/granite-4.0-3b-vision",
    "prompt": "Extract the table in this image to JSON.",
    "max_tokens": 1024,
    "temperature": 0.1
  }'
```

*(Nota: per chiamate multimodali complete con sglang dovrai passare l'immagine in base64 nel formato messaggi compatibile, questo è solo un esempio di ping)*.

## Domande frequenti

### Come funziona IBM Granite 4.0 3B Vision?
È un modello VLM che usa un LLM di base da 3.5B e un adattatore visivo LoRA da 0.5B. Elabora le immagini dei documenti e ne estrae dati strutturati come JSON o CSV.

### Posso usare Granite Vision 4.0 su CPU?
Sì, puoi usare framework come llama.cpp per eseguirne versioni quantizzate su CPU, sebbene le performance ottimali si ottengano con GPU o acceleratori come Apple Silicon.

### È adatto per la lingua italiana?
Sì. Granite è stato addestrato per essere multilingua, riuscendo a estrarre concetti e campi da documenti italiani, inglesi o francesi con ottima affidabilità.
